# 人性弱点利用与用户粘性策略

## 一、核心心理学原理

### 1.1 可变比率强化（Variable Ratio Reinforcement）

**原理**：奖励出现的不可预测性比固定奖励更能驱动持续行为（老虎机效应）。

**在小游戏中的应用**：
- 道具掉落概率随机（10%-40%），而非固定掉落
- 随机事件系统（19 种事件 × 3 种稀有度）
- Boss 波的出现间隔有微小随机扰动

```javascript
// ✓ 正确：可变概率
const dropChance = [0, 0.10, 0.20, 0.30, 0.40, 0.40]; // 按 Tier

// ✗ 错误：固定掉落
// 每击杀 5 个必掉 1 个 → 玩家很快失去期待感
```

### 1.2 损失厌恶（Loss Aversion）

**原理**：失去 100 元的痛苦是获得 100 元快乐的 2 倍。

**应用**：
- **分享复活**：死亡后「分享给好友可获 +1 命」— 玩家不愿失去已有进度
- **差一点失败**：展示「你曾占领 78% 的地图」— 放大「差一点就赢了」的损失感
- **连胜保护**：连胜 5 局后失败，显示「连胜中断！再来一局找回连胜」
- **巩固度系统**：占领区域越久防御越强，失去时损失更大

```javascript
// 失败弹窗的「差一点」数据 — 利用损失厌恶驱动重试
defeatInfo: function(level) { return '第 ' + level + ' 关'; },
// 配合显示：「再坚持一下就赢了！」「你曾消灭了 85% 的敌人！」
```

### 1.3 蔡格尼克效应（Zeigarnik Effect）

**原理**：未完成的任务比已完成的更令人念念不忘。

**应用**：
- **成就进度条**：显示「16/25 成就已解锁」— 不完整感驱动继续
- **关卡星级**：通关后显示 1/3 星 — 玩家想回来拿满星
- **角色解锁进度**：「再通过第 15 波即可解锁分身者」
- **选关地图**：60 关地图上已通关的明亮、未通关的暗淡

```javascript
// 成就系统 — 展示未完成项比已完成项更醒目
getCompletedCount: function() { ... },  // "16/25"
getTotalCount: function() { ... },
// UI 上未解锁成就用半透明图标 + 锁标记，制造不完整感
```

### 1.4 内在动机三要素（Self-Determination Theory）

| 要素 | 含义 | 游戏实现 |
|------|------|----------|
| **自主感** | 玩家觉得是自己在做选择 | 角色选择、策略路线、技能搭配 |
| **胜任感** | 玩家觉得自己在变强 | 连杀系统、成就解锁、难度递进 |
| **关联感** | 玩家觉得与他人有连接 | 排行榜、分享、昵称系统 |

### 1.5 沉没成本效应（Sunk Cost Fallacy）

**原理**：已投入的时间/精力使人不愿放弃。

**应用**：
- 成就系统的累计统计（累计击杀 500、累计拾取 100 个道具）
- 角色解锁是永久的 — 投入的时间不会白费
- 排行榜历史最佳记录 — 下次游戏从心理上延续上次的投入

## 二、粘性系统设计指南

### 2.1 成就系统（必做 ★★★）

成就是最低成本、最高回报的粘性系统。

**设计原则**：
1. **分类覆盖**：波次、击杀、连杀、特殊操作、综合 — 每类至少 3 个
2. **难度梯度**：每类内部从简单到困难递进（通过 5 波 → 10 → 15 → 20 → 25 → 30）
3. **关联奖励**：高难成就关联皮肤解锁（蔡格尼克效应 + 收集欲）
4. **即时公告**：解锁瞬间弹出成就公告（多巴胺即时反馈）

```javascript
// 成就定义模板
const ACHIEVEMENT_DEFS = [
  // 简单（80% 玩家可达成）
  { id: 'wave_5', name: '初出茅庐', desc: '通过第5波', icon: '🌱' },
  // 中等（30% 玩家可达成）
  { id: 'wave_15', name: '百战不殆', desc: '通过第15波', icon: '🏅' },
  // 困难（5% 玩家可达成）
  { id: 'wave_30', name: '终极试炼', desc: '通过第30波', icon: '🌋', skin: 'inferno' },
  // 隐藏（1% 玩家可达成）
  { id: 'all_achievements', name: '大满贯', desc: '完成所有其他成就', icon: '💎' },
];
```

**成就难度分布**：
- 简单（80%+ 达成率）：占总成就的 30%
- 中等（30-50% 达成率）：占总成就的 35%
- 困难（5-15% 达成率）：占总成就的 25%
- 隐藏/传奇（≤ 5% 达成率）：占总成就的 10%

### 2.2 排行榜系统（必做 ★★★）

**双排行榜设计**：
- **本地排行** — 个人历史最佳，零延迟，离线可用
- **全局排行** — 云端竞争，社交比较，每人最多 3 条记录

**关键设计**：
- 排行指标多维度（波次 + 击杀 + 连杀），避免单一指标
- 展示「我的排名」— 即使不在 Top 50 也能看到自己的位次
- 短 UID 展示（openid 后 6 位）— 有唯一感但保护隐私

### 2.3 角色/皮肤解锁系统（强烈推荐 ★★☆）

**角色系统设计**：
- 默认角色是全面型，后续角色各有特色（高风险高回报 / 坦克型 / 特殊技能型）
- 解锁条件关联游戏里程碑（通过 X 波、累计击杀 Y、达成 Z 连杀）
- 每个角色有独特视觉主题 + Emoji 图标

**皮肤系统设计**：
- 皮肤纯外观，不影响数值（公平性）
- 通过成就解锁（成就→皮肤绑定，双重激励）
- 每个皮肤有完整的配色方案（inner/outer/halo/trail/particle）
- 皮肤用动物主题 + 抽象绘制（不依赖图片）

### 2.4 分享复活（必做 ★★★）

**机制**：死亡后弹出「分享给好友可获 +1 命」，利用损失厌恶 + 社交传播。

**技术实现**：
```javascript
// 分享复活流程
wx.shareAppMessage({
  title: '我在刺客线条第' + wave + '波阵亡了，来帮我复仇！',
  // ... 分享参数
});
// 分享成功后
player.lives += 1;
player.invincibleEndTime = now + 2000; // 复活后 2 秒无敌
```

**规则约束**：
- 每局只能分享复活 1 次（防滥用）
- 复活后有 2 秒无敌保护（防复活即死）
- 分享文案包含当前进度（引发好奇心）

### 2.5 连胜/连杀系统（推荐 ★★☆）

**连杀系统**（局内）：
- 单次操作击杀多个敌人触发连杀
- 连杀称号递进：双杀 → 三杀 → 四杀 → 五杀 → 超神
- 连杀有专属音效（音调递增）和视觉公告
- 首次击杀触发「第一滴血」公告

**连胜系统**（跨局）：
- 连续通关累积连胜数
- 连胜有额外分数加成
- 连胜中断时用「差一点」话术驱动重试

## 三、首次体验优化（FTUE）

### 3.1 前 30 秒黄金法则

```
0-5 秒：   看到游戏画面，理解视觉风格
5-10 秒：  尝试第一次触控操作
10-20 秒： 获得第一次成功反馈（击杀/得分/占领）
20-30 秒： 理解核心循环，产生「我想再来一次」的冲动
```

### 3.2 新手引导原则

- **不用文字教学** — 通过关卡设计自然教会操作
- 第 1 波只有最弱的敌人（T1），数量少（2-3 个）
- 第 2 波引入新机制时，有明显的视觉提示
- 前 3 波/关是「保护期」— 难度远低于正常曲线

### 3.3 「差一点」失败设计

失败画面必须包含：
1. 本局最佳数据（最高连杀、最多击杀）
2. 与历史最佳的对比（「距离最佳记录还差 2 波！」）
3. 即将达成的成就进度（「再击杀 8 个即可解锁成就！」）
4. 大按钮「重新挑战」（颜色醒目，一键重试）
5. 小按钮「返回菜单」（不抢视觉焦点）

## 四、留存策略

### 次日留存（D1）目标 ≥ 30%

关键策略：
- 首次游戏结束时引导设置昵称（建立身份感）
- 展示全局排行榜位次（社交比较驱动）
- 展示未解锁的角色/皮肤（蔡格尼克效应）

### 7 日留存（D7）目标 ≥ 15%

关键策略：
- 成就系统的中等难度项需要 3-7 天达成
- 角色解锁需要累积一定游戏量
- 排行榜每周刷新（如有）

### 30 日留存（D30）目标 ≥ 5%

关键策略：
- 困难成就+传奇皮肤需要长期投入
- 困难模式专属成就
- 定期更新内容（新角色/新关卡/新事件）
