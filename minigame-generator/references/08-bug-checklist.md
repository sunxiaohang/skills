# 常见 Bug 检查清单（Top 30）

每次发布前、重大功能提交前逐项检查，标记 ✅ 或 ❌。

---

## A. 屏幕与渲染（1-8）

### 1. DPI 不匹配导致模糊/偏移
- [ ] Canvas 物理尺寸 = 逻辑尺寸 × dpr
- [ ] `ctx.scale(dpr, dpr)` 或 `ctx.setTransform` 正确设置
- [ ] 触摸坐标在逻辑坐标系中
- [ ] dpr 接近 1 时吸附到 1

### 2. 横屏旋转未完成时获取到竖屏尺寸
- [ ] `screenW = Math.max(windowWidth, windowHeight)`
- [ ] 始终保证 screenW ≥ screenH

### 3. safeArea 坐标系与屏幕方向不一致
- [ ] 检测 `isPortraitData = sw < sh`
- [ ] 竖屏数据时做坐标系映射
- [ ] 合理性校验（边距 > 15% 屏幕 → 回退 0）

### 4. 热启动 canvas 状态被系统重置
- [ ] `wx.onShow` 中强制刷新尺寸和 canvas
- [ ] 每帧开头 `syncCanvasDpr()` 确保变换矩阵正确

### 5. canvas.width 赋值被系统忽略（高性能模式）
- [ ] 设置后检查 `canvas.width` 是否与期望值一致
- [ ] 不一致时用 canvas 实际尺寸反推 dpr

### 6. 手动旋转模式下触摸坐标未映射
- [ ] `_needManualRotate` 时触摸坐标 `{ x: ty, y: physW - tx }`
- [ ] canvas 物理尺寸按竖屏设置

### 7. 绘制模糊（未对齐像素）
- [ ] 线条绘制偏移 0.5px（`x + 0.5`）使其对齐物理像素
- [ ] 文字坐标取整

### 8. 全屏清屏遗漏
- [ ] 每帧开头 `ctx.fillRect(0, 0, screenW, screenH)` 覆盖全屏
- [ ] 从后台切回后 canvas 可能变黑 → syncCanvasDpr 修复

## B. 输入与交互（9-14）

### 9. 多点触控 identifier 未正确追踪
- [ ] 用 `changedTouches`（不是 `touches`）
- [ ] 通过 `identifier` 区分不同触点
- [ ] touchend 时正确清理对应触点状态

### 10. 摇杆触摸区域超出底座但不响应
- [ ] 摇杆活动范围 = 底座半径 + 20px
- [ ] 摇杆球限制在最大偏移距离内

### 11. 死区抖动
- [ ] 摇杆死区 ≥ 8px
- [ ] 死区内输出方向为零向量

### 12. UI 按钮点击穿透到游戏层
- [ ] 弹窗/菜单显示时拦截游戏输入
- [ ] 按钮点击后设置冷却时间（150ms）防止重复触发

### 13. 双击误判（间隔过短）
- [ ] 双击间隔阈值 ≥ 300ms
- [ ] 检测两次点击位置在同一区域

### 14. 安全区域内的交互元素被遮挡
- [ ] HUD 元素偏移 `SAFE_LEFT/SAFE_TOP` 距离
- [ ] 右上角预留 90px 给小程序胶囊按钮

## C. 游戏逻辑（15-22）

### 15. deltaTime 异常大（标签页切换回来）
- [ ] `dt = Math.min(dt, 0.05)` 设上限
- [ ] 物理计算不会因大 dt 导致穿墙/瞬移

### 16. 碰撞检测遗漏（帧率低时穿过）
- [ ] 快速移动的小对象用射线/扫掠检测
- [ ] 碰撞半径用 `collisionScale`（0.85-0.9）微缩，体验更宽容

### 17. 对象移出世界边界
- [ ] 所有移动对象有边界 clamp
- [ ] 粒子边界用软碰撞（反弹 × 0.5）

### 18. 分裂/生成敌人在障碍物内
- [ ] 生成位置检查与现有障碍物的最小距离
- [ ] 分裂方向远离障碍物

### 19. 游戏结束后仍在处理输入/更新
- [ ] `gameState === 'gameover'` 时跳过游戏逻辑更新
- [ ] Game Over 后禁用摇杆

### 20. 无敌时间结束瞬间被秒杀
- [ ] 无敌结束后有 100-200ms 的碰撞冷却
- [ ] 受伤后有 300-500ms 的受伤无敌

### 21. 波次清空判定有残留
- [ ] 死亡动画中的敌人不计入存活数
- [ ] 假身（clone）不计入存活数

### 22. 数值溢出
- [ ] 兵力/分数有最大值限制（如 9999）
- [ ] 百分比计算避免除以零

---

## D. 存储与网络（23-26）

### 23. Storage 读写未 try-catch
- [ ] 所有 `getStorageSync` / `setStorageSync` 有 try-catch
- [ ] 读取失败返回默认值

### 24. 存档数据格式变更后旧存档崩溃
- [ ] 读取存档时逐字段校验，缺失字段用默认值
- [ ] 版本号标记（可选）

### 25. 云函数/API 调用无超时处理
- [ ] 网络请求设超时（5 秒）
- [ ] 失败时显示「加载失败，点击重试」

### 26. 排行榜重复提交
- [ ] 检查 wave+kills+multiKill 完全相同的记录
- [ ] 去重逻辑在服务端执行

---

## E. 音效（27-28）

### 27. 音效上下文创建失败导致崩溃
- [ ] `createWebAudioContext` 有 try-catch
- [ ] 上下文为 null 时静默返回

### 28. iOS 静音开关导致完全无声
- [ ] 启动时调用 `wx.setInnerAudioOption({ obeyMuteSwitch: false })`
- [ ] BGM `onError` 有容错处理
