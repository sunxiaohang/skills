# LongStory-Generator：长篇小说结构化生成器

**风格定位：搞笑无厘头 + 逻辑自洽 + 人物立体**

本 Skill 将长篇小说创作拆解为三个强制确认阶段，每阶段均有内部自检机制，确保风格一致、逻辑自洽、人物有血有肉、无遗漏。

---

## 输入格式要求

用户需提供（若未提供则主动询问）：

- **小说背景**：世界观、时代、核心冲突、风格基调（玄幻/悬疑/都市等）。
- **创作方向**：希望表达的主题、主角初步设想、预期篇幅或节奏（快节奏网文 / 慢热文学）。

收到用户输入后，直接进入第一步，无需额外确认。

---

## 第一步：生成故事创意梗概与核心框架

### 1.1 输出内容（总字数 ≤ 2000 字）

#### A. 故事创意梗概（300 字内）

- **一句话核心**：主角 + 核心冲突 + 独特无厘头亮点。
- **扩展段落**：世界观亮点、主角身份、核心冲突、结局暗示。
- **风格基调**：明确标注"搞笑无厘头，逻辑自洽型"，并给出 1-2 句风格示例。
  > 示例：主角是一个想当大侠的乞丐，但他唯一的武功是用脚趾头夹飞镖——不巧，这个世界的武功强弱取决于脚气浓度。

#### B. 核心框架（1500 字内）

- **三幕或起承转合结构**：每幕的核心功能、主要情节点、情绪曲线。
- **章节级大纲**：至少 10 个章节，每章包含：
  - 章节标题（暂定）
  - 核心事件（1-2 句话）
  - 结尾钩子类型（悬念 / 反转 / 情绪 / 任务 / 无厘头式意外）
- **关键冲突与转折点**：列出前 3 个主要转折点及对应章节位置。
- **主题与动机**：主角的核心欲望与核心恐惧，故事想探讨的母题。

### 1.2 内部自检（自动执行，不通过则拒绝进入下一步）

- [ ] 主角有明确的欲望和恐惧。
- [ ] 核心冲突具有可持续性（能支撑长篇）。
- [ ] 每个章节都有功能性（推进剧情或深化人物）。
- [ ] 钩子分布均匀，至少每 3 章有一个强钩子。
- [ ] 风格基调中至少包含一个"荒诞但可自圆"的示例。
- [ ] **主角的"优点"和"缺点"来自同一个性格根源**（已初步标注）。

### 1.3 用户确认

输出完成后询问：

> "请确认以上创意与框架。如需微调（如修改某章内容、改变结局方向），请指示。确认后进入第二步。"

## 第二步：生成关键上下文文档（故事圣经，约 1 万字）

### 2.1 输出内容结构（严格按以下模块，总字数 ≈ 1 万字）

#### 【模块 A】基础设定卡（约 1000 字）

- 世界观名称、核心法则（魔法、科技、社会制度等）。
- 时代背景（年份、纪年、历史大事件）。
- 地理与势力分布（地图简述、主要阵营）。
- 特殊名词/术语表（附解释）。
- 该世界的"荒诞规则示例"（至少一条无厘头但一致的设定）。

#### 【模块 B】人物档案（约 3000 字）

每个主要人物独立卡片，包含：

| 字段 | 说明 |
|------|------|
| 姓名、绰号、年龄、性别 | 基本信息 |
| 外貌特征 | 标志性穿着、疤痕、眼神等 |
| 性格关键词 | 3-5 个，**必须附带具体行为示例，包含至少1个"让读者皱眉但能理解"的行为** |
| 核心欲望 | "我想要……"——必须具体到一件事，不能是抽象词汇 |
| 核心恐惧 | "我害怕……"——必须具体到一件事，不能是抽象词汇 |
| **优缺点同根分析** | 该人物的优点和缺点如何来自同一个性格根源（见 B-补充规范） |
| 口头禅/习惯动作 | 可含无厘头特色 |
| 能力/技能等级 | 若适用 |
| 角色弧光 | 起始状态 → 变化触发事件 → 最终状态（见多卷弧光节点规范） |
| **搞笑属性** | 该人物的主要笑点来源 |
| **立体性种子** | 第一卷内埋入的1-2个"真实感细节"（见 B-立体性种子规范） |

次要人物简表：姓名 + 身份 + 一句话作用 + **一个让他不只是功能性存在的细节**。

---

**【B-补充规范：优缺点同根分析】**

脸谱化的本质不是"好人太好"，而是"这个人的所有行为只服务于一个功能标签，而不服务于他自己"。

每个主要人物的档案必须完成以下分析：

> **同根分析模板**：[性格根源] → 在顺境中表现为 [优点行为] → 在压力下表现为 [缺点/让人皱眉的行为] → 读者能理解但不一定认同。

示例：
- 陈默极度在意规则和流程 → 顺境：帮群众走绿色通道，对大爷大妈极有耐心 → 压力：用政务话术回避情感，把"好的您稍等"当成人际缓冲，实际是拒绝被人真正靠近。
- 林知夏责任感极强 → 顺境：判断迅速、执行力强、保护队员 → 压力：把所有人都当潜在危险品，无法在任务框架外表达关心，导致情感关系长期停在"工作模式"。

**禁止**：优点和缺点毫无关联（比如"他很善良，但是懒惰"——这两者没有内在联系，是贴标签而非立体）。

---

**【B-立体性种子规范】**

第一卷的主要任务是让读者认识并喜欢人物，不适合大量心理挖掘。但每个主要人物需要在第一卷内埋入 **1-2 个精准的"真实感细节"**，不展开、不解释，但让读者感觉"这个人背后有东西"。

立体性种子的类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| **反常动作** | 人物做了一个与人设不符的小动作，不解释 | 一直用政务话术的人，有一次沉默了三秒没有回答 |
| **不合时宜的执念** | 在危机中突然在意一件不重要的事，但那件事指向他真正的软肋 | 面对枪口第一反应是担心报表——报表不只是工作，是他用命换来的世界的证据 |
| **被戳中时的微小反应** | 平时极稳的人，某句话让他手停了一下 | 反派说"你守护的平凡会消失"，他没有反驳，只是眼神冷了 |
| **坏毛病但可爱** | 一个小缺点，让人觉得这个人是真实的 | 周远第一反应总是保护自己（椅子往旁边挪），但每次最后都会留下来 |

**每个主要人物的立体性种子必须在模块 B 中标注：埋设章节 + 种子内容 + 预期在哪一卷展开。**

---

#### 【模块 C】人物关系图谱（约 1500 字）

- 情感关系（爱情、友情、敌对、师徒、利益绑定等）
- 权力关系（上下级、制衡、背叛可能性）
- 历史纠葛（过去的恩怨、误会）
- 关键关系的事件锚点（例如"A 曾救过 B → B 对 A 绝对忠诚"）
- **无厘头关系点**（例如"C 和 D 是死敌，但因为都害怕主角的脚气而临时结盟"）
- **【新增】情感线里程碑规划**（见 C-情感线规范）

---

**【C-情感线规范】**

核心关系（尤其是主角与搭档/对立面的关系）需要规划**每卷一个情感里程碑**，确保情感推进有节奏、不突兀。

情感里程碑设计原则：
1. **不使用直白台词表达**：情感通过行动、物件、习惯变化体现，而非"我喜欢你/我信任你"。
2. **里程碑要与卷的主题挂钩**：情感推进和剧情推进同步，不单独存在。
3. **每个里程碑有一个"标志性场景"**：读者能在读完后明确感知关系发生了变化。

情感里程碑模板：

| 卷 | 里程碑名称 | 标志性场景描述 | 情感变化 | 表达方式（禁用直白台词） |
|----|-----------|---------------|----------|------------------------|
| 第一卷 | — | — | 从警惕到复杂信任 | 某个动作/物件 |
| 第二卷 | — | — | 从复杂信任到主动关心 | 某个非任务场景的出现 |

---

#### 【模块 D】故事详细脉络（约 3000 字）

按三幕/起承转合分段，每段内列出核心情节点、主角阻碍、配角高光时刻。

**分卷/分章细纲**：至少 20 个章节，每章包含：

| 字段 | 说明 |
|------|------|
| 章节编号与暂定标题 | — |
| 场景概要 | 何时、何地、何人、何事 |
| 本章目标 | 推动主线 / 揭示信息 / 人物成长等 |
| 结尾钩子 | 具体写法示例，可含无厘头型 |
| **人物弧光节点** | 若本章是某人物的弧光关键节点，标注：人物名 + 节点类型（见 D-弧光节点规范） |
| **风格提醒** | 若本章有特别离谱的笑点，提前标注其逻辑自圆方式 |

---

**【D-弧光节点规范】**

多卷长篇最常见的问题是：**配角在卷一被塑造后，后续卷中逐渐沦为工具人**。防止方法是为每个主要人物规划**跨卷弧光节点表**，确保每卷都有该人物的成长落点。

弧光节点类型：

| 节点类型 | 说明 | 触发条件 |
|----------|------|----------|
| **防守** | 人物处于舒适区，用惯有方式应对 | 卷初建立 |
| **动摇** | 某事件触碰到人物的核心恐惧或执念 | 卷中前段 |
| **失败/挫折** | 人物用惯有方式失败了，必须改变 | 卷中后段（每卷至少1次） |
| **重建** | 人物找到新的应对方式 | 卷末或下卷初 |
| **传承** | 人物把自己学到的东西给出去 | 终局卷 |

**强制要求**：
- 每个主要人物在每卷必须有至少 1 个弧光节点，并标注触发事件和对应章节。
- **"失败/挫折"节点不可缺失**：一个从未判断失误、从未付出代价的人物是脸谱化的。即使是主角，也必须在某个节点上"用惯常方式处理了，但这次失败了"或"正确处理了，但代价比预期更重"。
- 失败节点的设计原则：**失败要来自人物自身逻辑的局限，而不是外部因素的刁难**。

弧光节点汇总表（必须在模块 D 末尾附上）：

| 人物 | 卷一节点 | 卷二节点 | 卷三节点 | … | 终局落点 |
|------|---------|---------|---------|---|---------|
| 主角 | — | — | — | — | — |
| 配角A | — | — | — | — | — |

---

#### 【模块 E】框架结构拆解（约 800 字）

- 情绪节奏图（高潮与低谷分布）。
- **伏笔与呼应清单**（表格形式）：

  | 伏笔 ID | 埋设章节 | 回收章节 | 内容描述 | 中期铺设节点（每卷至少1次） | 笑点逻辑锚点（若有） |
  |---------|---------|---------|---------|--------------------------|-----------------|

  > **中期铺设强制要求**：跨多卷的重要伏笔，不能只在埋设章节和回收章节出现。每隔 1 卷必须有至少 1 次"侧面触碰"——以不同人物视角、不同物件或不同对话，轻轻提醒读者这条线还在，但不揭示答案。

- 时间线表格：关键日期/天数，标注事件。

#### 【模块 F】风格与一致性指南（约 700 字）

- **叙事视角（必须明确选定，不允许"建议"）**：
  - 推荐方案：**第三人称贴近视角（限制性全知）**——主视角跟随主角，保留内心吐槽，高光场景可做"小节内切换"（在同一章节内短暂切换至旁观者视角，前后仍回归主角视角）。
  - 若选第一人称：不得切换视角，高光场景用"别人的反应"折射主角的力量，而非直接描述。
  - **视角一旦确定，写作时不得临时更改**。

- 语气基调：搞笑、轻松、明快，带吐槽感。
- 禁止使用的词汇或表述（避免现代用语穿越到古代等）。
- 专有名词拼写统一规则。
- **人物信息输出规范**：人物出场时，禁止"清单式"连续输出4条以上信息（姓名+职位+年龄+能力+经历+性格判断）。改为：保留2-3条核心信息，其余通过后续章节的行为展示。

**F-4. 搞笑无厘头风格细则（强制）**

- 核心原则：荒诞但不失控，笑点服务于人物或情节，内部逻辑自洽。
- 具体手法（写作时至少选用 2-3 种）：
  1. **反逻辑但自圆**：看似离谱的行为，后续揭示其有一套歪理或特殊世界观设定。
  2. **夸张与反差**：严肃场景中突然插入荒谬细节，并立即给出合理原因。
  3. **吐槽与冷幽默**：角色对荒诞事件做理性分析，制造反差。
  4. **无厘头对话**：跳脱常规，答非所问但暗含逻辑。
  5. **意外翻转让严肃崩塌**：精心计划因极小概率的愚蠢原因失败。
- 禁止事项：为搞笑破坏核心设定、低俗笑点、笑点密度过高（每千字 2-4 个为佳）。

**F-5. "经得住推敲"的保障机制**

- 每章自检问题：
  - 每个"不合理"行为，在小说世界观内是否有逻辑链条？
  - 是否至少有一个笑点同时推进了情节或揭示了人物性格？
  - 有没有笑点与前面章节的设定矛盾？
- **笑点记录表**：记录每个笑点的触发方式、逻辑自圆句、所在章节。

**F-6. 人物立体性写作规范（强制）【新增】**

在生成正文时，以下规则强制执行：

**（一）主角不能总是对的**

主角每 5 章内必须出现至少 1 次以下情况之一：
- 判断有误，但以符合其性格的方式误判（不是外部刁难，是自身逻辑的盲区）
- 处理方式引发了意料之外的副作用（即使最终解决了，也有人受到情感伤害）
- 用惯常手段应对，发现这次不管用，需要调整

**（二）配角要有自己的逻辑**

配角出现时，他的行为必须首先服务于"他自己想要什么/害怕什么"，其次才服务于剧情推进。
检验方法：把配角从这个场景里拿走，他自己会去做什么？如果答案是"什么都不会做，因为他只在这个场景有意义"，则该配角脸谱化，需要修改。

**（三）反派/对立角色必须有完整的逻辑链**

反派的行动逻辑必须独立于主角存在。即：如果主角没有出现，反派会做什么？他的目标、手段、动机必须自洽，而不是"为了给主角制造阻碍而存在"。

推荐设计：**反派是主角的平行命运线**——相似的起点，不同的选择，相互照见。这比单纯的力量对立更有深度。

**（四）"好意造成的伤害"比主动作恶更有深度**

至少有一个主要冲突来源于：某人出于真实的善意做了某件事，但这件事对另一个人造成了伤害。双方都没有错，但都付出了代价。这是长篇小说区别于爽文的核心。

---

### 2.2 内部自检（自动执行）

- [ ] 每个主要人物都有完整的角色弧光。
- [ ] **每个主要人物的优缺点来自同一个性格根源（已完成同根分析）**。
- [ ] **每个主要人物在第一卷内有1-2个立体性种子（已标注埋设章节）**。
- [ ] **弧光节点汇总表已填写，每人每卷至少1个节点，含失败/挫折节点**。
- [ ] 人物关系至少有一个潜在冲突点。
- [ ] **核心情感关系已规划情感里程碑（每卷1个，标志性场景已描述）**。
- [ ] 伏笔清单中每个伏笔都标注了预期回收章节。
- [ ] **跨卷重要伏笔已规划中期铺设节点（每卷至少1次侧面触碰）**。
- [ ] 时间线无逻辑漏洞。
- [ ] 整份文档字数误差 ≤ 10%（9000-11000 字）。
- [ ] F-4、F-5、F-6 内容完整且可执行。
- [ ] **叙事视角已明确选定，不允许模糊表述**。

### 2.3 用户确认

输出完成后询问：

> "关键上下文文档已完成（约 1 万字）。请确认是否满足需要，或要求补充某模块（如增加配角、调整伏笔）。确认后进入第三步。"

## 第三步：生成小说正文

### 3.1 写作策略

- **分章生成**：每次只生成 1 章（约 2000-4000 字）。
- **前置记忆**：每次生成前自动注入：
  - 故事圣经核心要素（人物卡 + 关系 + 伏笔清单 + 风格细则）
  - 已生成章节的摘要列表（最近 3 章详细摘要）
  - 当前章节的大纲（来自第二步细纲）
  - **本章涉及人物的弧光节点状态**（当前处于哪个节点，下一个节点在哪里）

**指令模板（每章自动填充）：**

```
基于以下故事圣经：[插入第二步文档的关键摘要]
```

### 3.2 每章生成后的自检

- [ ] 主角名称、外貌、能力与人物卡一致。
- [ ] 所有专有名词拼写正确。
- [ ] 本章事件与时间线无冲突。

- [ ] **本章内主角至少有 1 次"不完全正确"的时刻（或标注说明为何本章豁免）**。
- [ ] **本章配角的行为有自己的逻辑，而非纯粹配合剧情**。

### 3.3 阶段性拼接与平滑

- **每 5 章**：自动执行一次一致性检查，输出报告：
  - 人物状态变化
  - 伏笔埋设与回收进度
  - 时间线核对
  - 笑点密度统计
  - **【新增】人物立体性检查**：
    - 各主要人物在近5章内是否出现过"让读者皱眉但能理解"的行为？
    - 各主要人物的弧光节点是否按规划推进？
    - 是否有人物开始呈现工具人倾向（行为只服务剧情，不服务自身逻辑）？

- **全文完成后**：执行最终平滑：
  - 合并所有章节为单个文本。
  - 检查章节间过渡句是否自然，为缺失处自动生成过渡段落。
  - 统一叙事视角与语气。
  - 生成**伏笔回收对照表**（埋设章节 → 中期铺设章节 → 实际回收章节 → 是否呼应）。
  - 生成**人物状态追踪表**（从第一章到最后的变化）。
  - 生成**笑点密度与逻辑自圆统计表**（每章笑点数量、最离谱情节及其解释章节）。
  - 生成**人物立体性评估报告**：
    - 各人物的优缺点同根性是否贯穿全文？
    - 各人物的弧光节点是否全部落实，是否存在某人物在某卷"消失"的情况？
    - 反派/对立角色是否保持了独立完整的行动逻辑？
  - 检查全篇是否有违背"经得住推敲"原则的笑点，如有则自动提示建议修改。

### 3.4 输出格式

- 每章单独输出，标题为"第 X 章 [章节名]"。
- 最终提供一个完整文本文件（或可复制的整合区）。

## 人物立体性快速参考卡

> 在任何阶段遇到"这个人物是否脸谱化"的疑问时，用此卡检验。

**检验问题一：这个人物有没有自己的逻辑？**
把他从当前场景拿走，他会去做什么？如果答案是"什么都不会做"，则工具人化，需要补充。

**检验问题二：他的优点和缺点是否同根？**
找到一个行为，它在某种情境下是优点，在另一种情境下是缺点。如果找不到，性格设计过于扁平。

**检验问题三：他有没有让读者皱眉但能理解的时刻？**
完美的人不真实。他必须在某处做了一个"不太对"但"符合他的逻辑"的选择。

**检验问题四：反派/对立角色是否是主角的平行线？**
相似的起点，不同的选择，相互照见——这是最有深度的对立设计。

**检验问题五：是否存在"好意造成的伤害"？**
双方都没错，但都付出了代价。这是长篇小说区别于爽文的核心张力来源。
