# LongStory-Generator：长篇小说结构化生成器（强钩子版）

**核心定位：搞笑无厘头 · 逻辑自洽 · 人物立体 · 每章强钩子 · 前5章留存率优化**

本 Skill 将长篇小说创作拆解为三个强制确认阶段，每阶段均有内部自检机制，确保风格一致、逻辑自洽、人物有血有肉、每章结尾留住读者。

---

## 底层逻辑（写作前必须内化）

### 搞笑无厘头爽文的留存公式

```
读者留存 = 笑点密度 × 钩子强度 × 人物辨识度
```

- 笑点密度不足 → 读者觉得"还行"，但不会追更
- 钩子强度不足 → 读者笑完就走，不翻下一章
- 人物辨识度不足 → 读者记不住，口碑无法传播

### 三类内容的比重控制

```
每章结构：
  前 1/3 → 建立现状或压力（慢节奏，铺垫）
  中 1/3 → 荒诞/冲突爆发（笑点释放 + 情绪推进）
  后 1/3 → 余韵 + 新悬念（情感落地 + 钩子引爆）
```

**铺垫章降级版**：
```
  前 1/2 → 日常荒诞积累（小笑点 + 世界观强化）
  后 1/2 → 主动行为 + 微钩子（角色在动，不是在等）
```

## 输入格式要求

用户需提供（若未提供则主动询问）：

- **小说背景**：世界观、时代、核心冲突、风格基调（玄幻/悬疑/都市等）。
- **创作方向**：希望表达的主题、主角初步设想、预期篇幅或节奏。

收到用户输入后，直接进入第一步，无需额外确认。

## 第一步：生成故事创意梗概与核心框架

### 1.1 输出内容（总字数 ≤ 2000 字）

#### A. 故事创意梗概（300 字内）

- **一句话核心**：主角 + 核心冲突 + 独特无厘头亮点。
- **扩展段落**：世界观亮点、主角身份、核心冲突、结局暗示。
- **风格基调**：明确标注"搞笑无厘头，逻辑自洽型"，并给出 1-2 句风格示例。
  > 示例：主角是一个想当大侠的乞丐，但他唯一的武功是用脚趾头夹飞镖——不巧，这个世界的武功强弱取决于脚气浓度。
- **标志性记忆点（预设）**：全书想要读者截图传播的核心台词/行为/物件（1个，见【H-4】）。

#### B. 核心框架（1500 字内）

- **三幕或起承转合结构**：每幕的核心功能、主要情节点、情绪曲线。
- **章节级大纲**：至少 10 个章节，每章包含：
  - 章节标题（暂定）
  - 核心事件（1-2 句话）
  - **钩子类型（必须标注，见【H-1】钩子分类系统）**
- **关键冲突与转折点**：列出前 3 个主要转折点及对应章节位置。
- **主题与动机**：主角的核心欲望与核心恐惧，故事想探讨的母题。

### 1.2 内部自检

- [ ] 主角有明确的欲望和恐惧。
- [ ] 核心冲突具有可持续性（能支撑长篇）。
- [ ] 每个章节都有功能性（推进剧情或深化人物）。
- [ ] **钩子类型已标注，且前5章每章均有强钩子（★★★★以上）**。
- [ ] 风格基调中至少包含一个"荒诞但可自圆"的示例。
- [ ] 主角的"优点"和"缺点"来自同一个性格根源（已初步标注）。
- [ ] **第1章第一句话已预设，通过"陌生人5秒测试"（见【H-2】）**。

### 1.3 用户确认

> "请确认以上创意与框架。如需微调（如修改某章内容、改变结局方向），请指示。确认后进入第二步。"

## 第二步：生成关键上下文文档（故事圣经，约 1 万字）

### 2.1 输出内容结构（严格按以下模块，总字数 ≈ 1 万字）

#### 【模块 A】基础设定卡（约 1000 字）

- 世界观名称、核心法则（魔法、科技、社会制度等）。
- 时代背景（年份、纪年、历史大事件）。
- 地理与势力分布（地图简述、主要阵营）。
- 特殊名词/术语表（附解释）。
- 该世界的"荒诞规则示例"（至少一条无厘头但一致的设定）。

#### 【模块 B】人物档案（约 3000 字）

每个主要人物独立卡片，包含：

| 字段 | 说明 |
|------|------|
| 姓名、绰号、年龄、性别 | 基本信息 |
| 外貌特征 | 标志性穿着、疤痕、眼神等 |
| 性格关键词 | 3-5 个，**必须附带具体行为示例，含至少1个"让读者皱眉但能理解"的行为** |
| 核心欲望 | "我想要……"——必须具体到一件事，不能是抽象词汇 |
| 核心恐惧 | "我害怕……"——必须具体到一件事，不能是抽象词汇 |
| **优缺点同根分析** | 优点和缺点如何来自同一个性格根源（见 B-补充规范） |
| 口头禅/习惯动作 | 可含无厘头特色；**标志性记忆点在此处标注** |
| 能力/技能等级 | 若适用 |
| 角色弧光 | 起始状态 → 变化触发事件 → 最终状态 |
| **搞笑属性** | 该人物的主要笑点来源（类型 + 示例台词/动作） |
| **立体性种子** | 第一卷内埋入的1-2个"真实感细节"（见 B-立体性种子规范） |

次要人物简表：姓名 + 身份 + 一句话作用 + **一个让他不只是功能性存在的细节**。

---

**【B-补充规范：优缺点同根分析】**

脸谱化的本质不是"好人太好"，而是"这个人的所有行为只服务于一个功能标签，而不服务于他自己"。

每个主要人物必须完成：

> **同根分析模板**：[性格根源] → 在顺境中表现为 [优点行为] → 在压力下表现为 [缺点/让人皱眉的行为] → 读者能理解但不一定认同。

示例：
- 陈默极度在意规则和流程 → 顺境：帮群众走绿色通道 → 压力：用政务话术回避情感，把"好的您稍等"当成人际缓冲，实际是拒绝被人真正靠近。

**禁止**：优点和缺点毫无关联（"他很善良，但是懒惰"——无内在联系，是贴标签而非立体）。

---

**【B-立体性种子规范】**

每个主要人物需要在第一卷内埋入 **1-2 个精准的"真实感细节"**，不展开、不解释，但让读者感觉"这个人背后有东西"。

| 类型 | 说明 | 示例 |
|------|------|------|
| **反常动作** | 人物做了一个与人设不符的小动作，不解释 | 一直用政务话术的人，有一次沉默了三秒没有回答 |
| **不合时宜的执念** | 在危机中突然在意一件不重要的事 | 面对枪口第一反应是担心报表——报表是他用命换来的世界的证据 |
| **被戳中时的微小反应** | 平时极稳的人，某句话让他手停了一下 | 反派说"你守护的平凡会消失"，他没有反驳，只是眼神冷了 |
| **坏毛病但可爱** | 一个小缺点，让人觉得这个人是真实的 | 第一反应总是保护自己，但每次最后都会留下来 |

**每个主要人物的立体性种子必须在模块 B 中标注：埋设章节 + 种子内容 + 预期在哪一卷展开。**

---

#### 【模块 C】人物关系图谱（约 1500 字）

- 情感关系（爱情、友情、敌对、师徒、利益绑定等）
- 权力关系（上下级、制衡、背叛可能性）
- 历史纠葛（过去的恩怨、误会）
- 关键关系的事件锚点
- **无厘头关系点**（例如"C 和 D 是死敌，但因为都害怕主角的脚气而临时结盟"）
- **情感线里程碑规划**（见 C-情感线规范）

---

**【C-情感线规范】**

核心关系需要规划**每卷一个情感里程碑**。

情感里程碑设计原则：
1. **不使用直白台词表达**：情感通过行动、物件、习惯变化体现。
2. **里程碑要与卷的主题挂钩**：情感推进和剧情推进同步。
3. **每个里程碑有一个"标志性场景"**：读者能在读完后明确感知关系发生了变化。

| 卷 | 里程碑名称 | 标志性场景描述 | 情感变化 | 表达方式（禁用直白台词） |
|----|-----------|---------------|----------|------------------------|
| 第一卷 | — | — | 从警惕到复杂信任 | 某个动作/物件 |
| 第二卷 | — | — | 从复杂信任到主动关心 | 某个非任务场景的出现 |

---

#### 【模块 D】故事详细脉络（约 3000 字）

按三幕/起承转合分段，每段内列出核心情节点、主角阻碍、配角高光时刻。

**分卷/分章细纲**：至少 20 个章节，每章包含：

| 字段 | 说明 |
|------|------|
| 章节编号与暂定标题 | — |
| **章节类型** | 铺垫章 / 标准笑点章 / 高潮章 |
| 场景概要 | 何时、何地、何人、何事 |
| 本章目标 | 推动主线 / 揭示信息 / 人物成长等 |
| **钩子类型与强度** | 见【H-1】，必须标注，铺垫章不低于★★★ |
| **笑点数量与类型** | 本章计划的笑点数（每千字2-4个）及主要类型 |
| **人物弧光节点** | 若本章是某人物的弧光关键节点，标注（见 D-弧光节点规范） |
| **风格提醒** | 若本章有特别离谱的笑点，提前标注其逻辑自圆方式 |

---

**【D-弧光节点规范】**

为每个主要人物规划**跨卷弧光节点表**，确保每卷都有该人物的成长落点。

| 节点类型 | 说明 |
|----------|------|
| **防守** | 人物处于舒适区，用惯有方式应对 |
| **动摇** | 某事件触碰到人物的核心恐惧或执念 |
| **失败/挫折** | 人物用惯有方式失败了，必须改变（每卷至少1次） |
| **重建** | 人物找到新的应对方式 |
| **传承** | 人物把自己学到的东西给出去（终局卷） |

**强制要求**：
- 每个主要人物在每卷必须有至少 1 个弧光节点。
- **"失败/挫折"节点不可缺失**：失败要来自人物自身逻辑的局限，而不是外部因素的刁难。

弧光节点汇总表（必须在模块 D 末尾附上）：

| 人物 | 卷一节点 | 卷二节点 | 卷三节点 | 终局落点 |
|------|---------|---------|---------|---------|
| 主角 | — | — | — | — |
| 配角A | — | — | — | — |

---

#### 【模块 E】框架结构拆解（约 800 字）

- 情绪节奏图（高潮与低谷分布）。
- **伏笔与呼应清单**（表格形式）：

  | 伏笔 ID | 埋设章节 | 中期铺设节点 | 回收章节 | 内容描述 | 笑点锚点（若有） |
  |---------|---------|------------|---------|---------|--------------|

  > **中期铺设强制要求**：跨多卷的重要伏笔，每隔 1 卷必须有至少 1 次"侧面触碰"。

- 时间线表格：关键日期/天数，标注事件。

#### 【模块 F】风格与一致性指南（约 700 字）

- **叙事视角（必须明确选定，不允许"建议"）**：
  - 推荐：**第三人称贴近视角（限制性全知）**——主视角跟随主角，保留内心吐槽，高光场景可做小节内切换。
  - 若选第一人称：不得切换视角，高光场景用"别人的反应"折射主角的力量。
  - **视角一旦确定，写作时不得临时更改**。

- 语气基调：搞笑、轻松、明快，带吐槽感。
- 禁止使用的词汇或表述（避免现代用语穿越到古代等）。
- 专有名词拼写统一规则。
- **人物信息输出规范**：人物出场时，禁止"清单式"连续输出4条以上信息。改为：保留2-3条核心信息，其余通过后续章节的行为展示。

**F-4. 搞笑无厘头风格细则（强制）**

- **底层结构**：所有笑点遵循"预期→违背→自圆"三步结构（详见【H-3】）。
- **五种危机中笑点形态**（详见【H-5】）：降级处理、行政化命名、物理知识应用、流程绑架、反向崇高。
- **写作时至少选用 2-3 种手法**：
  1. **反逻辑但自圆**：看似离谱的行为，后续揭示有一套歪理或特殊设定。
  2. **夸张与反差**：严肃场景中突然插入荒谬细节，并立即给出合理原因。
  3. **吐槽与冷幽默**：角色对荒诞事件做理性分析，制造反差。
  4. **无厘头对话**：跳脱常规，答非所问但暗含逻辑。
  5. **意外翻转让严肃崩塌**：精心计划因极小概率的愚蠢原因失败。
- **笑点节奏**：铺垫（1-3句）→ 笑点（1句）→ 余韵（1-2句）→ 继续推进。
- **密度控制**：每千字 2-4 个笑点；同类机制连续使用超过2次必须变形。
- **章节结尾笑点必须是钩子型**（想继续往下看），不能是收尾型。

**F-5. 人物立体性写作规范（强制）**

**（一）主角不能总是对的**

主角每 5 章内必须出现至少 1 次以下情况之一：
- 判断有误，但以符合其性格的方式误判（自身逻辑的盲区，不是外部刁难）
- 处理方式引发了意料之外的副作用
- 用惯常手段应对，发现这次不管用，需要调整

**（二）配角要有自己的逻辑**

配角出现时，他的行为必须首先服务于"他自己想要什么/害怕什么"，其次才服务于剧情推进。
检验方法：把配角从这个场景里拿走，他自己会去做什么？

**（三）反派/对立角色必须有完整的逻辑链**

反派的行动逻辑必须独立于主角存在。推荐设计：**反派是主角的平行命运线**——相似的起点，不同的选择，相互照见。

**（四）"好意造成的伤害"比主动作恶更有深度**

至少有一个主要冲突来源于：某人出于真实的善意做了某件事，但这件事对另一个人造成了伤害。双方都没有错，但都付出了代价。

### 2.2 内部自检

- [ ] 每个主要人物都有完整的角色弧光。
- [ ] **每个主要人物的优缺点来自同一个性格根源（已完成同根分析）**。
- [ ] **每个主要人物在第一卷内有1-2个立体性种子（已标注埋设章节）**。
- [ ] **弧光节点汇总表已填写，每人每卷至少1个节点，含失败/挫折节点**。
- [ ] 人物关系至少有一个潜在冲突点。
- [ ] **核心情感关系已规划情感里程碑（每卷1个，标志性场景已描述）**。
- [ ] 伏笔清单中每个伏笔都标注了预期回收章节。
- [ ] **跨卷重要伏笔已规划中期铺设节点（每卷至少1次侧面触碰）**。
- [ ] 时间线无逻辑漏洞。
- [ ] 整份文档字数误差 ≤ 10%（9000-11000 字）。
- [ ] F-4、F-5 内容完整且可执行。
- [ ] **叙事视角已明确选定，不允许模糊表述**。
- [ ] **【H-4】标志性记忆点已设计并填写三阶段表（主角核心台词/行为至少1个）**。
- [ ] **【H-2】第1章开篇第一句话已预设，通过"陌生人5秒测试"**。
- [ ] **细纲中每章钩子类型已标注，前5章钩子强度均为★★★★以上**。

### 2.3 用户确认

> "关键上下文文档已完成（约 1 万字）。请确认是否满足需要，或要求补充某模块（如增加配角、调整伏笔）。确认后进入第三步。"

## 第三步：生成小说正文

### 3.1 写作策略

- **分章生成**：每次只生成 1 章，**每章字数标准为 2500 字，硬性要求，不得低于 2300 字，不得高于 2800 字**。
  - 铺垫章目标字数：2300字（节奏轻快，不拖沓）
  - 标准笑点章目标字数：2500字
  - 高潮章目标字数：2500-2800字（爽感需要足够篇幅铺开）

- **前置记忆**：每次生成前自动注入：
  - 故事圣经核心要素（人物卡 + 关系 + 伏笔清单 + 风格细则）
  - 已生成章节的摘要列表（最近 3 章详细摘要）
  - 当前章节的大纲（来自第二步细纲）
  - **本章涉及人物的弧光节点状态**
  - **本章钩子类型预设（来自细纲）**

**指令模板（每章自动填充）：**

```
基于以下故事圣经：[插入第二步文档的关键摘要]
前情提要：[最近一章结尾的摘要]
请严格按照细纲生成第 X 章《章节标题》。

要求：
1. 主角 [名字] 的外貌/习惯需保持一致（如：他总是不自觉摸耳垂）。
2. 本章结尾必须使用钩子：[钩子类型 + 具体示例]，强度须达到 ★★★★。
3. 对话要体现 [人物 A] 的口头禅："XXXX"。
4. 不得提前揭露尚未到回收章节的伏笔（伏笔 ID：F-003 计划在第 8 章回收）。
5. 全文采用搞笑无厘头但逻辑自洽的风格：
   - 每千字包含 2-4 个笑点，笑点遵循"预期→违背→自圆"结构（见【H-3】）。
   - 每个笑点后必须有余韵（1-2句），不能笑完直接推进。
   - 禁止低俗笑点；所有笑点需能回溯到人物性格或世界设定。
6. 章节结尾笑点必须是钩子型（见【H-5】测试三），不能是收尾型。
7. 人物立体性检查（每章必做）：
   - [主角] 在本章是否有判断失误或惯常手段失效的场景？（若无，说明原因）
   - [本章出现的配角] 的行为是否首先服务于他自己的逻辑？
```

## 【冷启动专项规范】前5章生死关卡

> 前5章决定读者是否把书加入书架、是否形成追更习惯。此规范的优先级高于所有其他规范。

### 【关卡一】第1章：破冷启动

**目标**：读者读完第1章，心里有一个未解决的问题（"这个世界/这个人到底怎么回事？"）

**强制要求**：

#### ① 第一句话必须通过"陌生人5秒测试"（硬性红线）

> 把第1章第一句话发给一个没有看过本书的陌生人，对方在5秒内是否想知道"然后呢"？
> 不能通过 → 整句重写，不做局部修改。

**禁止使用的开篇句式**（AI味最重，直接pass）：
- "在这个……的世界里，……"
- "自从……以来，……"
- "他/她是一个……的人"
- "这一天，一切都不同了"
- "没有人知道，其实……"（用于第一句）
- "[姓名]，[年龄]岁，[职业/身份]。"（清单式介绍）

**推荐开篇结构**：
```
第1句：主角正在做的具体行为（动作，不是介绍）
第2-3句：让这个行为变得具体（数字/细节/感官描写）
第4句：闯入元素（与当前日常完全不搭的荒诞事物）
第5-6句：主角的第一反应（必须违背读者预期，符合人物逻辑）
段末：一个没答案的问题或行动悬念
```

示例（正确）：
> 我叫陈默，只想在政务服务中心摸鱼养老。
> 真的。

分析："真的"这两个字是第一个笑点——他在强调"真的"，说明有人/有事在质疑他这个愿望，悬念立刻出现。

#### ② 第1章前150字内必须出现荒诞事件或反常信号

读者看完150字，必须确认：**"这个世界/这个人有某种超出日常的东西"**。
- ✅ 主角独处时做了一个普通人绝对不会有的本能反应，不解释
- ✅ 世界观里的荒诞规则以日常方式出现，让读者会心一笑
- ❌ 150字内全是日常背景介绍，无任何荒诞/反常信号

#### ③ 第1章结尾必须是外部信号钩子，禁止内心感慨收尾

- ✅ 一个陌生的物件出现了，没有解释
- ✅ 有人说了一句话，意义不明，但读者感到"这件事没完"
- ❌ 主角望着窗外，感叹岁月如梭（内心感慨=没有"明天会发生什么"的推力）

### 【关卡二】第2-3章：建立荒诞世界观惯例

**目标**：让读者建立对这个世界"荒诞规则"的信任感，养成"这里的荒诞是有逻辑的"认知。

**第2章强制要求**：
- 必须有**目击者的荒诞事件**：不要求大场面，但必须有一个非主角的人**亲眼看见并有外化反应**。私下独自的荒诞不算——读者要看到有人被影响到。
- **本章使用【H-6】场景节拍模板B（日常荒诞积累型）写作**。

**第3章强制要求**：
- **第一个"确认型笑点"**：读者在此章明确感知"这是一部搞笑无厘头的书，而且笑点是有逻辑的"。
- 笑点必须有≥3人公开目击，且至少1人的具体反应写出来（否则等于没有围观，笑点无法"社交验证"）。
- **本章使用【H-6】场景节拍模板A（公开荒诞场景四节拍）写作**。
- 本章结尾：必须有"更大的荒诞/麻烦正在来"的具体外部信号。

**冷启动三章硬性红线**（违反任何一条，整章重写）：

| 红线 | 要求 | 检验方式 |
|------|------|---------|
| 红线① | 第1章必须有荒诞信号 | 读完第1章，能否描述出一件"普通世界不会发生的事" |
| 红线② | 第2章必须有目击者外化反应 | 有没有人对荒诞事件做出可见反应（不是内心困惑） |
| 红线③ | 第3章打脸/笑点有≥3人目击 | 场景中在场人数是否≥3人，且有具体反应 |
| 红线④ | 前3章主角必须有≥1次主动出手 | 主角有没有因为"自己想"而做的行动，而非被迫响应 |
| 红线⑤ | 笑点段落必须有情绪温度差 | 铺垫段和笑点段的节奏/句长/情绪是否有明显落差 |
| 红线⑥ | 第3章结尾必须有外部信号 | 结尾是否有具体的外部事件暗示"还没完" |

**目标**：让读者养成"这书每隔几章就有好笑的事"的追更预期。

**强制节奏**：

| 章节 | 内容重点 | 笑点类型 | 钩子强度 |
|------|---------|---------|---------|
| 第4章 | 铺垫新冲突 + 小笑点 | 人物习惯/世界观荒诞 | ★★★ |
| 第5章 | 第二个"确认型笑点"（中档） | 危机中荒诞（五种形态之一） | ★★★★ |

**第5章检验**：读者此时应该知道——
- ✅ 主角的荒诞逻辑/能力已经建立了惯例感
- ✅ 世界观的"荒诞规则"已经被读者接受为"这个世界的正常"
- ✅ 读者对主角有了"个性辨识度"（能说出一句主角专属的台词或行为）

## 【H-1】钩子分类系统（全书必用）

**钩子的效果 = 钩子质量 × 钩子位置**

### 钩子类型与强度评级

| 钩子类型 | 写法 | 适用位置 | 强度 |
|---------|------|---------|------|
| **后果前置** | 先写结果，再写原因（下章才解释） | 章节结尾 | ★★★★★ |
| **信息缺口** | 给出一半信息，另一半缺失 | 章节结尾/开头 | ★★★★ |
| **无厘头宣告** | 说出一句听起来荒诞但一定成立的话，不解释 | 段落开头/结尾 | ★★★★ |
| **行动悬念** | 主角做了一个让人看不懂的行动，不解释 | 章节开头 | ★★★ |
| **反常态细节** | 一个和场景完全不搭的具体细节，不解释 | 任何位置 | ★★★ |
| **对话切入** | 直接从对话开始，语境不明 | 章节开头 | ★★★ |
| **荒诞升级** | 刚解决一个荒诞问题，出现一个更荒诞的问题 | 章节结尾 | ★★★★★ |
| **身份/信息炸弹** | 某个细节暗示主角/世界比读者以为的更复杂 | 章节结尾 | ★★★★ |

**强制要求**：
- 每章结尾必须使用钩子，强度不低于 ★★★
- 前5章每章结尾钩子强度不低于 ★★★★
- **禁止用内心感慨收尾**：结尾的悬念必须来自外部事件（对话/物件/他人行动），不能是主角的内心独白

### 结尾意象连续性约束

每章结尾使用的具体意象须记录，**同一意象连续3章出现=违规**，须更换：

| 章节 | 结尾意象 | 连续出现次数 | 是否违规 |
|------|---------|-----------|---------|
| 第N章 | 月光/走廊 | 第2次 | ✅ |
| 第N+1章 | 月光/走廊 | 第3次 | ✅ |
| 第N+2章 | 月光/走廊 | 第4次 | ❌ 须更换 |

## 【H-2】开篇吸睛信息排列规范

**读者决定继续读的判断，在前 150 字内完成。**

### 章节开头三段检验规则（每章强制）

每章开头三段，必须回答以下问题：
1. 现在谁在哪里做什么？（场景锚定，≤2句完成）
2. 本章要打破的"当前状态"是什么？（张力来源）
3. 主角此刻最在意什么？（制造情感投入）

三个问题有任何一个未回答，开头须重写。

### 失败开篇对照

❌ 失败版（介绍式开篇）：
> 陈默，男，26岁，表面是政务服务中心的窗口办事员，实则是五年前拯救过世界的秩序天神。他平时很低调，只想准时下班，但这天，一切都变了……

问题：信息堆砌无画面；"但这天，一切都变了"是最烂的过渡句；主角还没行动，读者无情感投入；无任何笑点或钩子。

✅ 正确版（动作式开篇）：
> 我叫陈默，只想在政务服务中心摸鱼养老。
> 真的。

## 【H-3】笑点底层结构公式

**任何笑点都遵循同一个底层结构：预期 → 违背 → 自圆。**

缺少任何一环，笑点就会失效：
- 只有违背，没有预期铺垫：读者困惑，不是幽默
- 只有预期，没有违背：剧情正常推进，不好笑
- 违背之后没有自圆：读者出戏，"这不符合逻辑"

### 三步操作法

**第一步：建立预期**
预期来源：场景惯例 / 人物惯例 / 类型惯例

**第二步：违背预期**
必须符合**人物的内在逻辑**，而不是随机荒诞。
- 好的违背：来自人物的性格、执念、价值观
- 坏的违背：纯粹为了搞笑而搞笑（没有动机）

**第三步：自圆**
用一句话或一个动作解释为什么这个违背"在这个世界/这个人物身上"完全合理。**自圆句越简洁越好，不要解释过度。**

示例分析：

| 环节 | 内容 |
|------|------|
| 预期 | 特战队端枪冲进来，指控主角释放SSS级波动 |
| 违背 | 主角第一反应是担心报表有没有保存 |
| 自圆 | 那张表写了三小时，三百二十七项指标——世界都可以毁，表不能毁 |

❌ 失败版（无自圆）：
> 特战队冲进来，他没有害怕，而是在想今天晚饭吃什么。

❌ 失败版（自圆过度）：
> 他第一反应是担心报表。因为他是一个极度珍惜日常的人，他认为报表代表了世界正常运转，所以他宁可面对枪口也要先保存报表，这体现了他对平凡生活的执着……

### 笑点余韵规范

**每个笑点之后必须有1-2句余韵，不能笑完直接推进。**

余韵写法：
- 其他人物的反应（不是解释，是被影响）
- 环境的小变化（笑点落地之后的物理结果）
- 主角的内心补充（一句话，不能是解释，是他下一秒的想法）

❌ 没有余韵：
> 我把退回通知书贴在门框上。门关上了。

✅ 有余韵：
> 我把退回通知书贴在门框上。
> 门开始向内坍缩。
> 林知夏在我身后，沉默了三秒。
> 我听见她笔记本翻页的声音。
> 她大概在想，这件事写报告时怎么描述比较合适。

### 连续笑点变形规则

同一个笑点机制在一章内最多用 2 次，第 3 次必须变形：
- **升级**：同样的荒诞行为，在更严肃的情境下重复
- **接龙**：别人开始用同样的逻辑回应
- **倒置**：原本是A荒诞、B正常，变成B开始荒诞

## 【H-4】标志性记忆点设计机制

**最有传播力的作品，都有一个读者能单独截图的东西。**

特征：脱离上下文也能让人笑或被打动 / 在书里反复出现但每次都有新含义 / 最终在终局处完成升华。

### 三阶段设计（建立→变形→升华）

**第一阶段：建立（第一卷前5章）**
- 以"最功能性"的方式出现（看起来只是习惯或职业话术）
- 初次出现时读者笑了，但不觉得有深意
- 第一卷内至少出现 3 次，建立"惯例感"

**第二阶段：变形（中间各卷）**
变形方向：
- 场景升级（同样的台词，重量变了）
- 使用者改变（配角开始用同样的逻辑）
- 含义反转（原本是逃避的台词，开始变成主动承诺）

**第三阶段：升华（终局卷）**
最后一次使用，必须承载整本书的主题。读者读到这里，会想起前面所有出现过这个元素的场景，产生情感叠加。

示例（《秩序天神》"好的，您稍等"）：

| 阶段 | 出现情境 | 含义 |
|------|---------|------|
| 建立（第1章） | 面对持枪特战队，第一反应说"好的，您稍等" | 职业习惯，也是对麻烦的缓冲方式，读者觉得好笑 |
| 变形-1 | 面对梦魇污染时说同样的话 | 神明用行政话术处理宏大威胁，爽点来源 |
| 变形-2 | 这句话被当成他"不愿开口"的信号，配角能分辨他说这句话时是真稍等还是在逃避 | 关系深化的标志 |
| 升华（最后一章） | 回到03号窗口，对普通群众说"好的，您稍等" | 不再是话术，而是对世界的承诺：别急，秩序还在，一个一个来 |

**在故事圣经（第二步）中，标志性记忆点必须单独列出：**

| 元素 | 类型 | 建立章节 | 变形节点（每卷） | 升华章节 | 主题联系 |
|------|------|---------|----------------|---------|---------|
| [台词/行为/物件] | — | 第X章 | 卷二·第Y章 | 终局·第Z章 | — |

## 【H-5】笑点位置设计图与情绪温度规范

**笑点的效果 = 笑点质量 × 笑点位置。**

### 剧情区段与笑点类型对应表

| 剧情区段 | 笑点功能 | 推荐类型 | 禁止类型 |
|---------|---------|---------|---------|
| **铺垫区（危机前）** | 建立人物惯例，让读者熟悉角色的"正常反应" | 日常吐槽、习惯性怪行为、人物小执念 | 荒诞危机型笑点（消耗紧张感） |
| **危机前一刻** | 缓解焦虑，同时让笑点和即将爆发的危机形成反差 | 主角对危机的"行政化反应"、不合时宜的在意 | 冷幽默/长篇铺垫型 |
| **危机中** | 反差爆发，记忆点最强 | 用荒诞方式解决危机（见五种形态） | 纯吐槽（没有动作推进） |
| **危机后余波** | 帮读者卸力，同时埋下新问题 | 人物"假装没事"但细节露馅、荒诞总结 | 过度自我感动 |
| **日常回归段** | 重新锚定世界观，感受日常的价值 | 具体的烦恼（报表/打折菜）、普通人的真实问题 | 英雄感、大词 |

**黄金位置：章节第一句、危机中、章节最后一句**

这三个位置的笑点记忆度是其他位置的 3 倍。如果一章只能有 3 个笑点，必须覆盖这三个位置。

### 危机中笑点的五种形态（核心爽点来源）

1. **降级处理**：用比危机小几个量级的工具/方式解决
   > 示例：用订书钉封住梦魇污染 / 用行政退回通知书关闭旧日之门

2. **行政化命名**：给超凡事物贴上官僚标签，立刻祛魅
   > 示例：把怪物行为定性为"扰乱公共秩序·未备案·跨界入侵"

3. **物理知识应用**：用中学物理/化学逻辑压制超凡能力
   > 示例：燃烧三要素、电弧接地、冰锥相变

4. **流程绑架**：用现代行政流程"合法"阻止超凡行动
   > 示例：访客登记未通过 / 材料不齐退回原籍

5. **反向崇高**：在最不应该计较小事的时刻，主角在意的是最小的那件事
   > 示例：世界快毁灭时，主角最担心的是报表有没有保存

### 情绪温度切换规范

**搞笑文的情绪是"燃""爽""笑"。全程克制笔法是留存率杀手。**

```
铺垫段 → 冷（慢节奏、长句、日常感）
转折点 → 一句话（极短，独立成段）
笑点/爆发段 → 热（短句、具体反应、快节奏）
```

**"热"的写法规范**——翻盘/笑点发生后，必须出现以下至少两项：

| 热信号 | 示例 |
|--------|------|
| 短句独立成段（≤5字/句，连续3句） | "稳住了。""出血慢了。""她手在抖。" |
| 某人的具体身体反应 | 护士拿血压仪的手抖了，数字出来看了两遍 |
| 沉默的质量变化 | 不是"所有人沉默了"，而是"那个年纪大的护士碰了她一下" |
| 在场人物的即时语言 | 哪怕半句话，必须出来 |
| 环境细节的反差 | 刚才喧闹的房间忽然只剩仪器声 |

**禁止在笑点爆发段使用的写法**：

| 禁止写法 | 原因 |
|---------|------|
| "他云淡风轻地收好了……" | 主角淡定可以，但不能让整段都是淡定，要用围观者的失控来对比 |
| "某某若有所思地看着他" | 思考状态=情绪温度偏冷，笑点段需要更外化的反应 |
| "走廊里重新安静下来" | 安静描写适合铺垫，不适合笑点段——笑点后应该是"乱了"不是"静了" |
| 主角内心独白超过2句 | 笑点段主角视角越少越好，用外部视角的反应放大效果 |

## 【H-6】场景执行节拍模板

> **用途**：写章节前先对照场景类型，按节拍序列逐格填充，不再从原则反推。

---

### 模板A：公开荒诞场景四节拍（适用于公开笑点/打脸章）

**适用条件**：有明确荒诞/意外行为 + 旁人在场 + 触发者身份已知

```
节拍1【荒诞落地·压力建立】
  必须包含：
    - 触发事件的具体动作或台词（一句话，带具体物件或称呼）
    - 至少一个旁人的即时反应（笑/沉默/侧过脸），用具体行为写
    - 主角不立即反应，但有一个"留痕动作"（记住了/眼神变了/手停了）
  禁止：
    - 旁人完全没有反应
    - 主角无任何内部或外部留痕

节拍2【能力/荒诞行为展示·被动触发】
  必须包含：
    - 一个具体事件让主角不得不出手（不能是主角主动炫技）
    - 主角的动作描写：部位+角度+力道，至少一项精准细节
    - 事件结果用数字/状态量化（稳住了 / 少了一两八钱 / 三分钟）
  禁止：
    - 主角主动说"我来"（被动出手才有反差）
    - 结果只用"好多了""稳住了"等模糊词，不附具体数据

节拍3【见证者反应·认知升级】
  必须包含：
    - 见证者身份写明
    - 见证者的具体行为：不能只写"愣住了"，必须写他做了什么
    - 见证者的认知停顿：手顿/话断/物件停在半空
  认知深度选择（三选一）：
    - 一级：普通围观者，说不清楚，只是"那个人……怎么了？"
    - 二级：行业相关人员，认出能力，点出专业名词
    - 三级：特定知情者，认出身份或背景
  禁止：
    - 见证者只是"震惊"，无具体行为
    - 连续两章用同一级别见证者

节拍4【嘴替·荒诞者出丑】
  必须包含：
    - 嘴替角色：优先用"在场的旁观者或配角"而非主角本人
    - 嘴替台词：必须直接点名荒诞行为 + 反差
    - 荒诞触发者被迫表态：收起笑/放下东西/说一句不相关的话然后走
    - 主角不在场或不开口：嘴替效果由第三方完成
  禁止：
    - 嘴替由主角本人完成（主角直接反击≠嘴替）
    - 触发者只是"离开了"，没有被迫当众表态
```

### 模板B：日常荒诞积累型（三节拍）

**适用条件**：无大笑点，以世界观强化/人物习惯/悬念推进为主（铺垫章）

```
节拍1【世界观/人物惯例触点植入】
  方式选一：
    - 世界观荒诞规则的日常体现（它就这样，大家都习以为常）
    - 人物的荒诞执念出现，读者建立"他就是这样的人"
    - 前文钩子的后续信号（某物件出现/某人又出现了）
  必须有"留痕动作"：记下来/放进口袋/在本子上写一行
  禁止：整章无世界观荒诞信号且无前文悬念信号

节拍2【微笑点·小反差】
  触发条件：对方/世界"正常"运转的同时，主角的荒诞逻辑悄悄占了优势
  写法：
    - 世界/对方的"正常"行为
    - 主角的荒诞应对（精准量化，数字/物件/状态）
    - 对方在当时不知道自己已经在主角的逻辑里了
  外化旁人反应（可选，有则更强）：旁人的停顿或眼神变化

节拍3【钩子埋设·锁住期待】
  类型选一：
    - 荒诞升级型：刚解决一个荒诞问题，出现了更荒诞的问题信号
    - 身份型：有人在追踪主角，或某个秘密被逼近
    - 关系型：某段关系出现松动信号（皱眉/不说话/回头看）
  必须在最后1/4篇幅出现
  禁止：
    - 结尾只写日常收尾（关门/睡觉/出门），无任何悬念信号
```

---

### 模板C：克制型主角行动（三节拍）

**适用条件**：主角不直接反击，但行动本身完成了反击/推进

```
节拍1【对方出招·主角接招方式定性】
  对方行为：质疑/命令/试探，带具体物件或称呼
  主角接招方式（三选一）：
    - 不回答，行动代替话语
    - 回答一个数字/事实，不解释
    - 说一句话，对方不知道是什么意思，读者知道
  接招后主角"留痕"：写一行字 / 拿走一个物件 / 放回一个东西

节拍2【行动结果可视化】
  必须用可测量的单位呈现结果：
    - 数字：时间/金额/数量
    - 状态量：具体的结果描述（精准词汇）
    - 位置/物件变化：合上/放回/推到对方面前
  禁止：
    - 结果只用形容词描述（"非常准确""明显好多了"）
    - 没有任何外化动作，只有主角心理描写

节拍3【延迟反噬·事后回响】
  主角不在场时，对方的反应才完全暴露：
    - 对方单独的动作（放下杯子/收起笑/把东西推开）
    - 第三方的旁白（旁人的眼神/议论/反应）
  主角必须已经离开或者背对场景
  禁止：
    - 主角全程在场目睹自己的反噬（主角不能是反噬的直接见证者）
```

## 【H-7】高频套语禁用清单

**AI生成的行文在情绪过渡、节奏停顿、内心描写三个位置容易积累套语，密度超标后读者感到"这是程序跑出来的"。**

| 禁用写法 | 禁用原因 | 替代方向 |
|---------|---------|---------|
| **"停了一下"**（每章超过2次） | 通用节奏填充词，滥用后失去停顿感 | 用换行、短句、具体动作承载停顿；让停顿"可见"而非"被叙述" |
| **"有什么东西"**（每章超过1次） | 模糊情绪套语，用了10次后完全失去朦胧感 | 必须写出"那个东西"具体是什么，或直接用行为替代 |
| **"把X放进去/放在那里/收下"**（每章超过1次） | 动作化内心的写法有设计感，但连续使用后机械化 | 改为：直接写行为结果，或换用其他外化方式（沉默/转向/停止） |
| **"没有说话"作为段落结尾**（连续出现） | 裸用沉默连续出现后，沉默本身的重量消失 | 沉默必须附一个可见细节（手的位置/目光落点/呼吸变化） |
| **"低下头"+"抬起头"组合**（每章超过2次） | 人物动作库过窄，成为默认过渡动作 | 扩展动作库：转向/靠住/走开/拿起放下/停在原地/往旁边挪了一步 |
| **"他/她感到……"开头的内心描写**（每章超过3次） | 直接告知情绪，读者不需要体验 | 改为行为：不说"他感到紧张"，写"他把杯子放下了，手指停在杯壁上" |

每章生成后扫描，超限触发提示：
> "⚠️ 套语超限：'[具体词汇]' 本章出现[N]次，超出限额。请将超出部分替换为行为描写或换行短句处理。"

---

### 3.2 每章生成后自检

**【冷启动专项检查（仅第1-3章执行，优先级高于所有其他检查）】**

- [ ] 第1章：第一句话是否通过"陌生人5秒测试"（不能通过=整句重写）
- [ ] 第1章：前150字内是否出现荒诞信号（未出现=整章重写）
- [ ] 第1章：结尾是否为外部信号钩子（内心感慨=重写结尾）
- [ ] 第2章：是否有目击者亲眼看见荒诞事件并有外化反应（未出现=整章重写）
- [ ] 第3章：公开笑点/荒诞场景是否有≥3人在场，且至少1人具体反应写出来（未达标=补写）
- [ ] 前3章内：主角是否有≥1次主动出手（非被叫去/被打电话的被动响应）
- [ ] 第3章笑点/爆发段：是否出现了≥2个"热信号"（短句/身体反应/环境反差/即时对话）
- [ ] 第3章结尾：是否有具体外部信号暗示"更大的荒诞/麻烦还在来"

**【常规检查（每章执行）】**

- [ ] 字数是否达到本章节类型的目标范围（铺垫章≥2300字 / 标准笑点章≥2500字 / 高潮章≥2500字）
- [ ] 打脸/笑点段落情绪温度是否与铺垫段有明显落差（热信号是否出现≥2个）
- [ ] 章节结尾钩子类型与细纲一致，强度≥★★★（前5章须≥★★★★）
- [ ] **结尾是否为外部信号钩子（禁止内心感慨收尾）**
- [ ] **结尾意象是否连续使用超过3章（超过=本章强制更换意象）**
- [ ] 每千字笑点数量是否在2-4个之间
- [ ] 每个笑点是否遵循"预期→违背→自圆"结构
- [ ] 每个笑点后是否有余韵（1-2句），未直接推进剧情
- [ ] 本章笑点机制是否有连续使用超过2次未变形的情况
- [ ] 未出现设定文档之外的"即兴设定"（如有，提示用户决定是否保留）
- [ ] **人物出场时未做清单式信息堆砌（单次出场信息量≤3条）**
- [ ] **本章内主角至少有1次"不完全正确"的时刻（或标注说明为何本章豁免）**
- [ ] **本章配角的行为有自己的逻辑，而非纯粹配合剧情**
- [ ] **开头三段是否通过"场景锚定/张力来源/情感投入"三问检验**
- [ ] **【H-5】危机中笑点是否使用了五种形态之一，章节最后笑点是否为钩子型**
- [ ] **【H-7】高频套语检查：各禁用词是否在限额内**
- [ ] **【H-4】本章是否有"可传播的台词或画面"（截图给陌生人能感到"这人/这世界有意思"）**

### 3.3 阶段性拼接与平滑

- **每 5 章**：自动执行一次一致性检查，输出报告：
  - 人物状态变化
  - 伏笔埋设与回收进度
  - 时间线核对
  - 笑点密度统计
  - **钩子强度分布（前5章是否均≥★★★★）**
  - **人物立体性检查**：
    - 各主要人物在近5章内是否出现过"让读者皱眉但能理解"的行为？
    - 各主要人物的弧光节点是否按规划推进？
    - 是否有人物开始呈现工具人倾向？

- **全文完成后**：执行最终平滑：
  - 合并所有章节为单个文本。
  - 检查章节间过渡句是否自然，为缺失处自动生成过渡段落。
  - 统一叙事视角与语气。
  - 生成**伏笔回收对照表**。
  - 生成**人物状态追踪表**。
  - 生成**笑点密度与逻辑自圆统计表**（每章笑点数量、最离谱情节及解释）。
  - 生成**人物立体性评估报告**：
    - 各人物的优缺点同根性是否贯穿全文？
    - 各人物的弧光节点是否全部落实？
    - 反派/对立角色是否保持了独立完整的行动逻辑？
  - 生成**全书钩子效果报告**：
    - 统计各章章末钩子强度，低于★★★的章节列出并提供修改建议
    - 统计各章可传播笑点数量——每章≥1个，否则标红提示
    - 标记各类笑点机制累计使用次数，超过3次未变形的标记
  - 生成**【H-4】标志性记忆点升华确认表**：
    - 建立章节 → 变形节点 → 升华章节，三阶段是否完整？

### 3.4 输出格式

- 每章单独输出，标题为"第 X 章 [章节名]"。
- 最终提供一个完整文本文件（或可复制的整合区）。

## 无遗漏保障机制

- 每一步都有强制检查清单，内部自动验证，若检查不通过则拒绝进入下一步并给出修正建议。
- 第二步的上下文文档包含风格与一致性指南，写作时持续引用。
- 每章生成前重新加载最近 3 章的摘要，避免跨章矛盾。
- **冷启动专项规范的优先级高于所有其他规范**：前5章违反红线，整章重写，不做局部修改。
- 最终整体做三次扫描：逻辑一致性扫描 + 风格一致性扫描 + 人物立体性扫描。

## 人物立体性快速参考卡

> 在任何阶段遇到"这个人物是否脸谱化"的疑问时，用此卡检验。

**检验问题一：这个人物有没有自己的逻辑？**
把他从当前场景拿走，他会去做什么？如果答案是"什么都不会做"，则工具人化，需要补充。

**检验问题二：他的优点和缺点是否同根？**
找到一个行为，它在某种情境下是优点，在另一种情境下是缺点。如果找不到，性格设计过于扁平。

**检验问题三：他有没有让读者皱眉但能理解的时刻？**
完美的人不真实。他必须在某处做了一个"不太对"但"符合他的逻辑"的选择。

**检验问题四：反派/对立角色是否是主角的平行线？**
相似的起点，不同的选择，相互照见——这是最有深度的对立设计。

**检验问题五：是否存在"好意造成的伤害"？**
双方都没错，但都付出了代价。这是长篇小说区别于爽文的核心张力来源。

---

## 钩子快速参考卡

> 在任何章节结尾遇到"这个钩子够不够强"的疑问时，用此卡检验。

**测试一：陌生人30秒测试**
把这章结尾的最后3句话发给一个没看过本书的人，对方是否想知道"然后呢"？
- 想知道 → 钩子有效
- 不想知道 → 检查：结尾是内心感慨还是外部信号？改用外部信号。

**测试二：可传播性测试**
把本章最好笑的一句话单独截图，发给没有看过本书的陌生人，对方能在5秒内感受到有意思吗？
- ✅ 能 → 这是可传播的笑点，是本书的名片
- ⚠️ 需要上下文 → 不算失败，但不是传播型笑点
- ❌ 完全不懂 → 检查是否依赖过多设定积累，考虑重写

**测试三：钩子效果测试**
读者笑完/读完这章，下一秒想做什么？

| 读者反应 | 评级 |
|---------|------|
| 立刻翻下一章（因为好奇接下来怎么样） | ★★★★★ |
| 往上回看（确认刚才是不是真的这么荒诞） | ★★★★ |
| 满足地笑完，然后可以停下来了 | ★★★ |
| 觉得好笑但不知道为什么好笑 | ★★ |
| 没有反应 | ★ |

**强制要求：每章结尾的最后一个笑点/钩子，必须是★★★★以上。前5章必须是★★★★★。**
