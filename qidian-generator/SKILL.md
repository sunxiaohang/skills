# Qidian-Generator：起点网风格长篇小说生成器

**核心定位：严肃世界观 · 王者成长弧 · 长线伏笔 · 付费订阅留存 · 适合起点/纵横/掌阅**

本 Skill 专为起点主流爽文设计，区别于番茄短平快的流量打法，起点读者是**付费长期追更**的群体——他们买的是"沉浸感 + 成长快感 + 史诗兑现"。本 Skill 的每一条规范都服务于这个核心目标。

---

## 【规则优先级总表】（冲突时按此顺序执行）

> 本 Skill 所有规则不等价。当两条规则冲突时，高优先级规则完全覆盖低优先级规则，不做折中。

```
优先级层级（从高到低）：

  P0【绝对铁律】> P1【读者情绪保障】> P2【结构规则】> P3【风格建议】
```

| 优先级 | 类型 | 包含规则 | 冲突处理 |
|--------|------|---------|---------|
| **P0 — 绝对铁律** | 不可违反，违反整章重写 | 首订红线①-⑤（【Q-1】）、钩子强度底线（每章≥★★★）、连续铺垫≤3章、低谷期≤5章 | 无论任何理由，一律重写 |
| **P1 — 读者情绪保障** | 读者感受 > 结构完整性 | 战斗情绪债（战前积累不足=整段重写）、低谷禁用人格崩塌、打脸前屈辱积累充分 | 结构可以调整，读者情绪不能将就 |
| **P2 — 结构规则** | 建议性强制，允许例外但须标注 | 五段式战斗结构、三幕节奏、伏笔触碰频率、扰动卡牌使用 | 有充分剧情理由时可偏离，但须在快照中记录 |
| **P3 — 风格建议** | 执行有助于质量提升，但非硬性 | G-6四项反AI感指令（频率见分层规则）、闲笔注入、微小视角类型选择 | 允许当章豁免，但连续3章豁免须警告 |

**首订红线覆盖一切**：第1-3章期间，P0红线优先于任何用户要求或剧情需要。若用户要求与首订红线冲突，须先提示用户，再执行。

## 底层逻辑（写作前必须内化）

### 起点读者心理模型

```
起点读者的留存公式：

  订阅转化 = 世界观投入感 × 主角成长快感 × 长线悬念牵引
  月票续费 = 爽点兑现节奏 × 读者情感绑定 × 社区互动感
```

**三个核心区别**（对比番茄）：

| 维度 | 番茄读者 | 起点读者 |
|------|--------|--------|
| 消费方式 | 免费刷，像刷短视频 | 付费追，像追美剧 |
| 容忍铺垫 | 极低（2章没爽就走） | 较高（愿意等剧情兑现） |
| 核心诉求 | 密集即时刺激 | 世界观投入 + 成长爽感 + 史诗感 |
| 讨论行为 | 基本不讨论 | 书评区/贴吧大量讨论、理论研究 |
| 追更周期 | 短（几天） | 长（数年） |
| 幽默期待 | 无厘头搞笑密集 | 主角高冷毒舌、偶发反差幽默 |

### 起点爽文核心公式

```
爽感来源 = 主角碾压感 + 世界观沉浸感 + 打脸快感

其中：
  碾压感 = 成长速度 × 金手指质量 × 实力对比反差
  沉浸感 = 体系自洽度 × 细节密度 × 读者参与感（书评区讨论）
  打脸快感 = 屈辱积累 × 翻盘力度 × 围观人数
```

### 章节节奏模型

```
起点章节结构（每章3000-4000字）：

  前 1/4 → 承接上章钩子 + 建立本章张力
  中 1/2 → 主线推进（成长/冲突/信息揭露）
  后 1/4 → 爽点兑现 + 新钩子引爆

爆发章（打脸/突破/史诗大战）：
  前 1/3 → 积压屈辱或危机（读者憋着气）
  中 1/3 → 翻盘过程（爽感释放）
  后 1/3 → 余威展示 + 更大布局开启
```

## 第一步：生成故事梗概与核心框架

### 1.1 输出内容（总字数 ≤ 2500 字）

#### A. 故事创意梗概（400字内）

- **一句话卖点**：主角身份反差 + 金手指类型 + 核心爽点方向
  > 示例："被废丹田的宗门弃徒，觉醒万古第一炼丹系统，从此每炼出一枚丹药，整个大陆的炼丹师都要怀疑人生。"
- **扩展段落**：世界观亮点、主角起点悲惨程度（屈辱感积累）、核心冲突、结局暗示
- **风格基调**：明确标注（严肃热血 / 克制毒舌 / 轻松升级 / 史诗大气），并给出1-2句风格示例
- **首订钩点（预设）**：读者看完第3章会订阅的理由是什么？（见【Q-1】）

#### B. 核心框架（1800字内）

- **三幕结构**：每幕的核心功能、主要情节点、情绪曲线
  - 第一幕（冷启动卷）：建立同情 + 金手指激活 + 首次小翻盘
  - 第二幕（成长主线）：多级打脸梯队 + 体系深化 + 大势力冲突
  - 第三幕（史诗收尾）：终极对决 + 世界观揭秘 + 主题兑现
- **章节级大纲**：至少15个章节，每章包含：
  - 章节标题（暂定）
  - 核心事件（1-2句）
  - **章节类型**：铺垫章 / 小爽章 / 大爽章 / 史诗章
  - **钩子类型（必须标注，见【Q-2】）**
- **打脸梯队预设**（见【Q-3】）：至少3级梯队，每级敌人简述
- **金手指初步设计**：类型 + 核心能力 + 上限暗示 + 可钻研性

### 1.2 内部自检

- [ ] 主角起点足够悲惨/屈辱（读者能产生同情心，后续翻盘才有爽感）
- [ ] 金手指有独特性，且有"可钻研性"（读者能讨论它的上限和规则）
- [ ] 核心冲突能支撑数百章（不是一场战斗就能解决的矛盾）
- [ ] 打脸梯队已有3级以上预设
- [ ] 第3章能给出"首订钩点"（见【Q-1】）
- [ ] 风格基调明确，且不是无厘头搞笑（起点主流不是这个赛道）
- [ ] 世界观有"体系感"（读者能感受到背后有完整规则，而非随意设定）

### 1.3 用户确认

> "请确认以上创意与框架。如需调整（修改金手指设计、改变梯队结构、调整风格基调），请指示。确认后进入第二步。"

## 第二步：生成故事圣经（约1.2万字）

### 2.1 输出内容结构（严格按以下模块，总字数 ≈ 1.2万字）

---

#### 【模块 A】世界观与体系设计（约2500字）

> 起点读者会在书评区写"体系分析帖"，世界观必须经得起读者反复推敲。

**A-1. 世界观基础**
- 世界名称、纪年、历史大事件
- 地理格局（大陆/城市/势力分布，附简单示意）
- 该世界的"底层法则"（修炼之道/超自然来源/科技逻辑）

**A-2. 核心体系设计（重点）**

修炼/能力体系必须包含：

| 要素 | 说明 | 设计要求 |
|------|------|---------|
| **等级划分** | 明确的境界/等级（至少8级） | 名字要有代入感，不能是"一级/二级" |
| **等级对应能力** | 每级能做什么，有哪些代表性战斗能力 | 要具体，不能只说"很强" |
| **突破条件** | 如何晋级，有无门槛/代价 | 要有挑战性，让读者理解"为什么主角能突破而别人不能" |
| **等级天花板感知** | 读者能感知到最强等级有多强 | 早期要给"仰望"，后期要有"我也到了"的爽感 |
| **金手指如何打破规则** | 主角的金手指如何在体系内占优势 | 要有"作弊感"但不能无脑，要有代价或条件 |

**A-3. 势力格局**
- 主要势力（3-5个）：名称 + 在体系中的地位 + 与主角的关系
- 隐藏势力/秘密组织（为后期伏笔预留）
- 主角出身的势力及其在格局中的位置（越弱越好，翻盘反差大）

**A-4. 金手指完整设计（核心模块）**

```
金手指设计四要素：

  ① 核心能力：主角独有的、别人没有的能力是什么
  ② 使用条件：需要满足什么条件才能使用（代价/资源/时机）
  ③ 成长路径：金手指本身如何随剧情解锁/升级
  ④ 可钻研性：体系内有哪些规则，读者能推测、讨论、预判
```

**强制要求**：
- 金手指不能是"无限制的万能"，必须有**内在规则**（读者才能参与讨论）
- 金手指的成长路径要与剧情推进深度绑定（不是自动升级，是靠剧情解锁）
- 书评区"体系分析帖"测试：如果有人写"主角金手指原理分析"，他能有东西可写吗？

**A-5. 特殊名词表**（至少10条，格式：名词 → 定义 → 在故事中的功能）

---

#### 【模块 B】人物档案（约2500字）

**主角卡片（必须完整填写）**

| 字段 | 说明 |
|------|------|
| 姓名、年龄、起点身份 | 越屈辱越好（废柴/弃徒/赘婿/孤儿等） |
| 外貌特征 | 起点常用：平凡外表隐藏强大气质，或一个不为人知的特征 |
| 性格关键词 | 3-5个，重点：**冷静、克制、毒舌（偶发）、对敌无情、对自己人有底线** |
| 核心欲望 | 具体到一件事（复仇/成为强者/保护某人/找出真相） |
| 核心恐惧 | 具体到一件事（失去/变成某种人/某件事重演） |
| **王者气质设计** | 见 B-王者气质规范 |
| 标志性行为/台词 | 主角的"招牌动作"，让读者一看到就知道是他（见【Q-4】）|
| 成长路径规划 | 卷一到终局的境界变化 + 每次突破的剧情锚点 |
| **屈辱积累设计** | 第一卷积累的屈辱（至少3件），对应后期的打脸章节 |

---

**【B-王者气质规范】**

起点主角不是番茄的"运气好的普通人"，他有**王者气质**——但要**克制展示**，不是一开始就嚣张。

王者气质的三种表现方式：
1. **面对威胁时的反应异于常人**：别人会害怕的东西，他不怕——不是因为无知，是因为见过更强的
2. **毒舌反击是克制的、精准的**：一句话让对方无法反驳，但不是破口大骂
3. **对自己的目标有绝对的执念**：外人觉得不可能，他淡淡地说一句，然后去做了

**禁止设计**：
- ❌ 主角刚登场就嚣张（没有铺垫的嚣张让读者反感）
- ❌ 主角靠运气而不是靠实力/谋略翻盘
- ❌ 主角对所有人都傲慢（冷傲vs对己方人的人情味，这是反差）

---

**【B-屈辱积累规范】**

起点打脸爽的前提是：**屈辱要先积累到位。**

屈辱积累设计原则：
- 屈辱来源必须**有名有姓**（不是模糊的"众人嘲笑"，而是"张三当众说了什么"）
- 屈辱场景要有旁观者（越多越好，打脸时他们都要在场）
- 屈辱积累时读者要有"憋着气"的感觉，不是无聊地看主角被欺负

屈辱积累检验：读者看完屈辱场景，心里是否出现了"迟早要让你好看"的念头？

---

**配角设计要求**

| 类型 | 设计要求 |
|------|---------|
| **盟友/跟班** | 不能是纯工具人，要有自己的成长弧，要有让读者喜欢的特质 |
| **女主/感情线** | 起点感情线是长线投资，不要快进，里程碑要和剧情同步 |
| **反派（各梯队）** | 见【Q-3】反派设计规范 |
| **长辈/师傅型** | 要有自己的秘密，不能只是主角的工具人 |

次要人物简表：姓名 + 身份 + 功能 + **一个让他不只是功能性存在的细节**

---

**【B-配角成长弧规范】**

> 起点顶流书的配角往往有自己的书评区粉丝群。配角的存在感不是靠"帮主角办事"堆出来的，而是靠**他自己在追求某件事**。

**核心配角必须填写成长弧档案**（至少2名配角）：

| 字段 | 说明 |
|------|------|
| 配角姓名 | — |
| 独立目标 | 他/她在追求什么？这个目标和主角主线可以并行但不依附 |
| 成长阶段 | 每卷他/她有什么变化（不一定是实力，可以是认知/立场/关系） |
| 与主角的依存关系 | 他帮主角，主角也帮他——要双向，不能单向输出 |
| 死亡/离场预案 | 若此配角计划离场：何时？以何方式？死得是否"值得"？ |
| 书评区吸引点 | 这个配角有没有让读者想在书评区讨论他的特质或谜题 |

**配角死亡设计规则**：
- 配角死亡必须在故事圣经中预先规划，不能随机处理
- 死亡前必须有1-2章的"情感积累"（让读者更在乎他）
- 死亡的意义要明确：推动主角成长 / 兑现某个伏笔 / 改变主线走向
- ❌ 禁止：毫无铺垫地突然死亡 / 死亡章节草草处理 / 死后主角两章就忘了

---

**【B-灵魂注入：反套路拷问】（强制执行）**

> 完成基础人物档案后，必须针对**主角**和**核心反派/配角**各回答以下题库中的3-5题。
> 答案不直接写入故事，而是作为人物的"隐藏底色"，影响其在关键时刻的行为逻辑。
> 如果回答这些问题时感到"这个问题好难回答"——说明你还没真正认识这个人物。

**主角专属题库**

1. **他/她最难以启齿、自己都不愿承认的一个弱点是什么？**
   ——不是"太重感情"这种表面缺点，而是触及人性深处的东西。
   示例："他其实很庆幸仇人死了，这让他意识到自己和仇人的区别，比想象中小得多。"

2. **他/她会在什么情况下，为一个自己鄙视的人拼命？**
   ——这个情况是什么？他为什么没法拒绝？他事后怎么看待这件事？

3. **当他/她最终坐上王座，内心深处最怀念的，反而是哪个失去的、不值一提的小东西？**
   ——不是某个人，是某个具体的、微不足道的事物或时刻。

4. **除了主线目标，他/她有什么"不务正业"却无比痴迷的小爱好？**
   ——这个爱好必须和他的"强者人设"有反差，且有来历。
   示例："一个绝世剑仙，最大的爱好是偷偷在后山种菜，因为那是他童年里唯一由自己掌控的一块土地。"

5. **他/她有没有一件事，做了之后从来不提，但如果有人提起，他会沉默？**
   ——那件事是什么？他为什么不提？沉默的背后是什么？

**反派/核心配角专属题库**

1. **这个反派做过的、最让主角（或读者）不得不尊敬的一件事是什么？**
   ——不是"他其实有苦衷"这种洗白，是真实的、有骨气的行动。

2. **他/她有没有在某个深夜，为自己做过的事感到过一丝真实的后悔？**
   ——是因为什么？这种后悔有没有改变他后来的行为？还是他压下去了？

3. **他/她和主角之间，最核心的区别只有一个——那个区别是什么？**
   ——起点最好的反派是主角的"平行命运线"：相似的起点，一个不同的选择，走向了完全不同的终局。

**执行规则**：
- 主角：至少回答4题
- 核心反派（第3梯队以上）：至少回答2题
- 回答必须具体到细节，禁止空泛回答（"他有一颗善良的心"不算）
- 灵魂拷问答案必须在故事圣经中单独列出，并标注"将在哪个章节/场景中以行为方式体现"

---

#### 【模块 C】人物关系图谱（约1000字）

- 势力归属关系（各角色属于哪个势力）
- 情感关系（爱情、友情、仇恨、利益绑定）
- **隐藏关系**（为后期信息炸弹预留，注明计划揭露时间）
- 关键关系的事件锚点（哪件事改变了关系）
- **感情线里程碑规划**（每卷一个，见 C-感情线规范）

---

**【C-感情线规范】**

起点感情线是长线"糖"，不要急于兑现，要让读者长期期待：

| 卷 | 里程碑名称 | 标志性场景 | 关系变化 | 表达方式 |
|----|-----------|-----------|---------|---------|
| 第一卷 | 初次刮目相看 | — | 从轻视到复杂注意 | 行动（不是台词） |
| 第二卷 | 主动靠近 | — | 从注意到主动关心 | 一个非剧情需要的出现 |
| 第三卷 | 患难时刻 | — | 从关心到信任 | 将背后交给对方 |

**强制原则**：
- 感情推进要和剧情推进同步，不能为了糖而糖
- 起点读者喜欢"死要面子但行动出卖了他"的表现方式
- 女主不能太早成为主角的人（读者要长期期待）

#### 【模块 D】故事详细脉络（约3000字）

按三幕分段，每段内列出核心情节点、主角阻碍、打脸时机、大场面节点。

**分卷/分章细纲**：至少30个章节，每章包含：

| 字段 | 说明 |
|------|------|
| 章节编号与暂定标题 | — |
| **章节类型** | 铺垫章 / 小爽章 / 大爽章 / 史诗章 / 水章（需注明控制方式）|
| 场景概要 | 何时、何地、何人、何事 |
| 本章目标 | 成长推进 / 打脸 / 世界观深化 / 伏笔埋设 / 感情推进 |
| **钩子类型与强度** | 见【Q-2】，必须标注 |
| **打脸梯队标注** | 本章涉及哪级打脸对象，积累屈辱还是兑现打脸 |
| **境界/成长节点** | 本章是否有突破，突破的剧情锚点是什么 |
| **月票爆发节点** | 若本章是月末/重要节点，是否规划为大爽章 |

---

**【D-打脸梯队规范（见【Q-3】展开）】**

细纲中必须标注每章属于哪个打脸周期：

```
打脸周期 = 屈辱积累（3-5章）→ 实力展示（1-2章）→ 打脸兑现（1章）→ 余波（1章）
```

**强制要求**：每个打脸周期内，屈辱积累章节数≥打脸兑现章节数。不能刚屈辱一章就马上打脸——憋得不够，爽感打折。

---

**【D-剧情扰动骰子】（大爽章/史诗章前必须执行）**

> 严格的"屈辱-蓄力-打脸"循环，读多了读者会预判你的预判，爽感变成程序化作业。
> 每一个规划中的大爽章或史诗章之前，**必须从以下卡牌中抽取1张**，强制将该扰动融入该章的细纲。
> 这些扰动不是"让主角变弱"，而是让计划的爽点"不那么顺利地到来"——意外，才是"神展开"的来源。

**扰动卡牌库（每次大爽章/史诗章前，随机或选择1张）**：

---

🃏 **【意料之外的盟友】**
在计划打脸的场景中，突然出现一个之前被主角得罪过的小人物，提供了意想不到的关键帮助——但他的动机并非善意，而是另有目的。
→ 主角因此欠下一笔新的情感债，或种下一个新的隐患。

---

🃏 **【对手的尊严】**
在计划让对手彻底崩溃时，对手却展现出了意料之外的骨气或底线，让主角（和读者）在爽感之余，心头微微一震。
→ 打脸仍然完成，但读者对这个反派多了一点复杂的情感。下一级梯队的设计可以因此更有张力。

---

🃏 **【代价浮现】**
在打脸成功的瞬间，主角同时得知一个相关的沉重代价——他的行动无意中伤害了一个无辜的人，或付出了他当时没有意识到的东西。
→ 爽感不消失，但余波里多了一层无法言说的重量。这是"灵气"的来源之一。

---

🃏 **【非理性的人味】**
在绝对理性的计划中，主角因为一个极度私人的、非理性的原因（如：对手手腕上的一个胎记，让他想起了死去的弟弟）而临时改变了行动，导致计划外的结果。
→ 结果可以是好的，也可以是让事情变得更复杂的。但主角的行为在这一刻，第一次不服务于目标，而服务于他自己。

---

🃏 **【见证者的意外反应】**
打脸场景的旁观者中，有一个人的反应完全不在预期之内——他没有震惊，没有折服，而是做了一个让主角看不懂的举动。
→ 这个旁观者由此成为一个悬念，可以是未来的盟友、对手，或一个长线伏笔的起点。

---

🃏 **【胜利之后的空洞】**
打脸完全成功，主角得到了想要的一切。但在人群散去的那一刻，他突然意识到：这件事他等了很久，但真正到来的时候，感觉和想象中不一样——他不知道那个"不一样"是什么。
→ 这一刻不需要解释，不需要独白，只需要一个微小的动作或停顿。读者会自己感受到。

---

**执行规则**：
- 每张卡牌只影响当前这一个大爽章/史诗章的细纲设计，不影响全局走向
- 扰动内容需要在细纲中标注（"本章使用扰动卡：[卡牌名称]，融入方式：[简述]"）
- 禁止因为扰动让主角本次失败（扰动不是让主角输，是让胜利的路上多了一个转折）
- 连续两个大爽章不能使用同一张卡牌

**【扰动卡牌章节类型限用表】**

> 某些卡牌会破坏特定章节类型的核心情绪，使用前必须核对。

| 卡牌名称 | 禁用章节类型 | 禁用原因 |
|---------|-----------|---------|
| 🃏 **代价浮现** | **大爽章** | 大爽章的核心功能是密集爽感释放，代价会稀释爽感，读者会感到被克扣 |
| 🃏 **胜利之后的空洞** | **史诗章** | 史诗章是全书情绪最高点，空洞感会损伤史诗感兑现，令读者对长期投入感到后悔 |
| 🃏 **对手的尊严** | 第1-2梯队打脸章 | 早期反派不需要复杂化，过早引入悲悯情绪会干扰屈辱-爽感的纯粹循环 |
| 🃏 **非理性的人味** | 战略推进章（主角需要做出关键决策的章节）| 关键决策章节若主角因非理性原因行动，读者会质疑主角智商 |

**禁用不等于永远不用**：禁用的是在该类型的**当前章节**使用，在其他类型章节仍可正常使用。

---

#### 【模块 E】伏笔与长线架构（约1000字）

起点读者的粘性来自"跨越数百章的伏笔兑现"，这是平台差异化的核心。

**长线伏笔分类**：

| 类型 | 伏笔描述 | 埋设章节 | 中期触碰节点 | 兑现章节 | 兑现方式 |
|------|---------|---------|------------|---------|---------|
| **身份类** | 主角真实身世/隐藏身份 | 第X章 | 卷二·第Y章 | 卷N·第Z章 | — |
| **世界观类** | 世界运行的真正规则 | — | — | — | — |
| **仇恨类** | 仇人的真实面目 | — | — | — | — |
| **感情类** | 感情线的重要转折 | — | — | — | — |
| **体系类** | 金手指的真正上限 | — | — | — | — |

**强制要求**：
- 重要伏笔每隔1卷必须有至少1次"侧面触碰"（让读者感知到它还在，没有被作者忘记）
- 身份类伏笔是起点标配，必须有
- 体系上限的伏笔要做到"读者自己能推测但推测不准"

---

**【E-情感债系统】**

> 情感债不是剧情信息，而是角色之间一种**未被言明、未被偿还的情感负担**：愧疚、误解、隐忍的牺牲、深埋的感激。它让感情线不再是"建立关系-里程碑-结局"的打卡流程，而是一张充满张力的、动态的人情债网络。

**情感债登记表**（故事圣经中必须填写）：

| 债务编号 | 欠债方 | 债主 | 债务内容 | 欠债原因 | 计划偿还卷次 | 偿还方式（禁用：直接告白/解释） |
|---------|------|------|---------|---------|-----------|--------------------------|
| EQ-001 | — | — | — | — | — | 一个具体行动 |

**情感债的"偿还"设计规则**：
- 一笔情感债的偿还，**禁止通过直白台词完成**（"我一直愧疚你的死"不算偿还）
- 偿还必须通过一个**具体的、非言语的行动**体现——这个行动本身就是最好的剧情点
- 偿还不等于"清账"：有些债偿还后会产生新的债

示例：
> 欠债：主角因一个决定，导致曾经救过他的一个同门战死。他从未提起过这件事。
> 禁止偿还方式：某天主角对那人的妹妹说"你哥哥的死是我的责任"。
> 正确偿还方式：多年后，主角在一场他本可以不参与的战役里，孤身挡在那人妹妹前面——他没有解释，事后也没有提。那人妹妹看着他的背影沉默了很久。

**每5章一致性检查中，增加情感债盘点**：
- 主角当前欠谁的债？本卷有没有（有意或无意地）偿还了一部分？
- 谁欠了主角的债？这笔债有没有因新的事件加重或减轻？
- 本卷新增了哪笔情感债？（每一次"打脸成功"后，主角都应该检查：有没有意外欠下什么？）

- 水章控制方案（哪些章节是铺垫，如何在水章中埋小钩子）
- 时间线表格：关键日期/天数，标注重要事件

#### 【模块 G】风格与一致性指南（约600字）

- **叙事视角**（必须选定）：
  - 起点主流：**第三人称有限视角（贴近主角）**，主角视角下的世界，高光时刻可借助旁人视角放大
  - 第一人称：适合系统流/日记体，节奏更快，但难做史诗感
  - **视角一旦确定，不得临时更改**

- **语气基调**：起点主流偏严肃热血，幽默是偶发的克制型
- 专有名词统一规则
- 章节标题风格（起点常用：悬念型 "第X章 XXX来了？" / 动作型 "第X章 出手" / 信息型 "第X章 XXX的真相"）
- 禁止使用的词汇（避免现代网络用语穿越进古代/玄幻世界）

**G-5. 起点风格写作细则（强制）**

（一）**克制型幽默写法**（不是无厘头，是毒舌 + 反差）：
- 主角在面对嘲讽时不直接反击，说一句"是吗"然后用行动打脸
- 系统/金手指本身的梗（播报战力数据时的荒诞落差）
- 旁观者的反应才是幽默的主体（见证者目瞪口呆，主角淡淡走开）

（二）**大场面写法规范**：
- 大战前必须有3章以上铺垫（力量对比 + 利害关系 + 读者情感投入）
- 大战中不能一笔带过，分解为"危机 → 应对 → 反转 → 余波"四步
- 大战结束必须有"震慑性余波"（对手彻底服气 / 围观者的认知更新 / 更大势力注意到主角）
- **【余波结束必须执行：微小视角收束】**（见下方规则，强制）

**【微小视角收束规则】**

> 千军万马的史诗大战、万众瞩目的打脸结束后，最后一个镜头，**必须从一个极其微小的、容易被忽略的视角切入**。
> 这是"灵气"的来源——当所有人的目光聚焦在舞台中央时，你的笔触落在了无人注意的角落。

每次大场面的余波段，从以下类型中**随机或选择一个**执行：

| 视角类型 | 写法示例 | 效果 |
|---------|---------|------|
| **无名旁观者的眼睛** | 一个普通士兵/平民，在历史性的一刻，只想着如何活着回家 | 让宏大叙事有了人的温度 |
| **不起眼的物件** | 战场上一朵被踩碎的野花 / 主角衣袖上被划破的一个口子 / 反派座椅上陈旧的划痕 | 用具体细节承载无法言说的情绪 |
| **感官记忆的触发** | 大战结束后，空气里经久不散的铁锈味，让主角想起了七岁那年父亲铁匠铺里的一个午后 | 把史诗时刻和个人命运连接，产生叠加情感 |
| **对手最后的微小动作** | 彻底落败的反派，最后低头看了一眼手上的某样东西，然后闭上了眼睛 | 在胜利的高光里，埋下悲悯的余温 |
| **身边最不重要的人的反应** | 所有人在欢呼，但一个平时说话最多的小跟班，这次一句话都没说 | 反差放大情感，让沉默比欢呼更有力量 |

**执行规则**：
- 大场面的余波段字数不超过300字，微小视角收束占其中的1/3到1/2
- 微小视角收束之后，禁止再有任何宏大叙事句子（比如"这一战，永载史册"）
- 微小视角的选择须与本章主角的"灵魂拷问"答案有内在呼应（如果主角的拷问答案里提到了父亲的铁匠铺，那感官记忆类型就是首选）

（三）**【G-5-战斗写作完整规范】**

> 战斗不精彩的根本原因不是「招式不够多」，而是**读者没有理由在乎这场打架**。
> 战斗描写的本质是情绪管理：让读者先紧张，再释放，再留下余震。
> 以下规范适用于全书所有战斗场景，按战斗规模分三级执行。

---

**【战斗情绪债：战斗前的必要积累】（所有战斗章必须执行）**

> 读者对这场战斗有多在乎，决定了战斗有多精彩。
> 战斗本身是情绪的释放——但释放之前，必须先有积累。

战斗前必须完成以下至少2项：

| 积累类型 | 操作方式 | 效果 |
|---------|---------|------|
| **仇恨绑定** | 对手是之前屈辱过主角的人，或伤害过主角在意的人 | 读者主动想看他输 |
| **旁观者替读者担心** | 围观者低声说"他不可能赢" / 队友劝主角别打 | 读者心里开始替主角捏汗 |
| **主动放弃退路** | 主角本可以走，但他没有走 | 读者感受到主角的决意，情绪跟着上来 |
| **利害关系明确** | 这场战斗输了会发生什么，赢了能得到什么——要具体，不能是抽象的"尊严" | 读者知道为什么要在乎这场打架 |
| **实力差距可见** | 通过旁观者反应/上一场战斗的结果，让读者感知对手有多强 | 逆境感建立，翻盘才有爽感 |

**红线**：战斗前积累不足 = 战斗再精彩也爽感打折，整段重写，不是修改。

---

**【战斗节奏五段式结构】（所有战斗必须遵守）**

```
① 开局定性（10%字数）
   目的：让读者知道「这是一场硬仗」
   操作：主角一上来处于劣势，或对手先展示一个让旁观者噤声的动作
   禁止：主角一开场就压制对手（没有紧张感，后续无从翻盘）

② 第一次应对（20%字数）
   目的：建立「主角在动脑子」的认知
   操作：主角用一个意料之外但事后理所当然的方式撑住了第一波
   禁止：主角靠运气躲过，或完全被动挨打没有任何应对

③ 中场压力加码（25%字数）
   目的：让读者以为要输了
   操作：对手变招，主角第一个应对被克制，重新陷入危机
         ——这是全段最紧张的部分，字数可以放慢，每一句都在加压
   禁止：危机太短、太轻，读者还没来得及担心就过去了

④ 核心反转（35%字数）
   目的：爽感释放
   操作：主角展示「他一直在等这个时机」的反转手段
         ——金手指首次真实出场 / 提前布局的陷阱收网 / 身体潜能突破极限
   关键：反转要有铺垫（哪怕只是一句话），让读者回想起来说「他早就埋好了」
   禁止：反转从天而降，没有任何前文依据

⑤ 余震（10%字数）
   目的：爽感延续 + 下章钩子
   操作：见下方「战斗余震四选一」规则
   禁止：「他赢了」一句话收场，或主角自我感慨收尾
```

---

**【战斗余震四选一（强制）】**

每场战斗结束，必须从以下类型中选择至少一个执行：

| 余震类型 | 写法 | 作用 |
|---------|------|------|
| **旁观者认知更新** | 刚才说「他不可能赢」的人，现在的表情/沉默/一个动作 | 打脸完成信号，旁观者越多越爽 |
| **对手的最后反应** | 落败一方的最后一个细节（见【微小视角收束规则】）| 给战斗留余温，让爽感不那么空洞 |
| **更大威胁的注意** | 这场战斗引来了不该注意到主角的人/势力 | 格局扩张钩子，为下一级打脸梯队埋线 |
| **主角身上的代价** | 赢了，但留下了一个小代价（伤/消耗/暴露了什么）| 让胜利有重量，防止战斗变成无成本爽文 |

---

**【战斗感官描写规范】（每场战斗必须覆盖3种以上感官）**

> 战斗描写最常见的AI感 = 全是视觉 + 抽象形容词。
> 疼痛是触觉，喘息是听觉，血腥气是嗅觉——这些才是「临场感」的来源。

| 感官 | 禁止写法 | 正确写法示例 |
|------|---------|-----------|
| **触觉（最重要）** | 「他感到一阵剧痛」 | 「那一拳打在左肋，他听到了一声闷响，像是什么东西错位了——疼是后来的事，先来的是一种奇怪的麻」 |
| **听觉** | 「声音震耳欲聋」 | 「那声响不大，像一根湿木头被折断，但周围突然安静了」 |
| **嗅觉** | 「空气中弥漫着血腥气」 | 「他嘴里有股铁腥味——不知道从什么时候开始的，也许是咬破了腮帮子」 |
| **视觉（克制用）** | 「速度快得难以捕捉」 | 「他的眼睛跟上了，但脚还没动——等脑子把指令传下去，人已经到了」 |
| **内感（最易被忽略）** | 「他感到一阵恐惧」 | 「他的呼吸浅了，不是因为累，是胸口有个地方在收紧，像被什么攥住了」 |

**【按战斗规模分级写法】**

**小型战斗（1v1，无大量旁观者，铺垫章/小爽章）**

- 字数控制在600-1000字，不要贪多
- 核心在「一个关键动作的细节」，不需要完整的五段式
- 必须有：1个感官细节 + 1个旁观者反应（哪怕只有1人）+ 1个钩子
- 结束方式：对手态度变化比「他赢了」更有说服力

**中型战斗（有围观、有情绪积累，小爽章/大爽章）**

- 完整执行五段式结构
- 旁观者戏份要有2-3次：「赛前担心」→「战中惊呼」→「结束后沉默」
- 必须有嘴替角色：替读者说出他们想说的话
- 结束必须用「战斗余震四选一」

**大型战斗/史诗战（多方势力，大爽章/史诗章）**

- 大战前3章以上铺垫：力量对比 + 利害关系 + 读者情感投入三件套
- 战斗内部再拆分为「小高潮序列」：每1000字一个小起伏，不允许平铺
- 必须有主角的「绝境时刻」：读者必须真实地以为要输了
- 必须执行「微小视角收束规则」（见上方）
- 大战结束必须引出更大格局

---

**【战斗节奏的「加速与减速」控制】**

> 战斗最忌「匀速」——每一招都用同等笔墨，读者看到后来已经麻木。
> 快慢对比，才是战斗节奏的灵魂。

**加速写法**（用于「压力加码」和「反转瞬间」）：
- 句子极短，独立成行
- 停顿极少，动作接动作
- 省略心理描写，只有动作和感官

示例：
> 他没来得及想。
> 脚往旁边移了一步。
> 那掌风擦过耳边——头发都跟着动了。

**减速写法**（用于「绝境时刻」和「余震」）：
- 句子拉长，带大量感官细节
- 在一个关键动作上停住，把那一秒拉成半页
- 主角的内感：不是情绪台词，是身体的反应

示例：
> 他靠在那面墙上，听见自己的心跳。
> 不是很快——反而慢得不正常，一下，一下，像有人在数节拍。
> 他知道那是什么。是身体在告诉他，已经到极限了。
> 他低头看了一眼右手——手指还在，但感觉不到了。

---

**【战斗常见问题快速诊断】**

| 读者反馈 | 根本原因 | 修复方式 |
|---------|---------|---------|
| 「打得不激烈」 | 感官描写全是视觉+抽象词 | 加触觉/听觉/内感，把关键动作放慢 |
| 「不紧张」 | 旁观者没有担心，读者没有情绪积累 | 战斗前补「旁观者担心」+ 「主动放弃退路」 |
| 「不爽」 | 反转缺乏铺垫，或余震太弱 | 检查反转是否有前文依据，补充旁观者认知更新 |
| 「太长了」 | 匀速平铺，没有快慢对比 | 压缩中段，把字数集中在「加码」和「反转」 |
| 「读完没感觉」 | 没有代价，没有遗留悬念 | 加「主角身上的代价」余震 + 结尾钩子 |
| 「对手太弱」 | 反派威胁感建立不足 | 回查【Q-3附】反派威胁感建立协议 |

（三）**升级节奏控制**：
- 每3-5章有一次明显的"变强信号"（不一定是境界突破，可以是技能解锁/战力认可/旁人态度变化）
- 突破前必须有"卡关"（主角遇到了只能靠更强实力解决的问题）
- 突破过程要有画面感，不能是"一夜之间突破了"

**【G-6. 反AI感写作指令】（每章生成时强制执行）**

> AI生成文字的本质缺陷是"过于流畅"——句式整齐、情绪到位、一路推进，读起来没有阻力，但也没有质感。
> 以下四条指令，是专门用来制造"阻力"的。各指令执行频率不同，见下方分层表。

**【G-6 执行频率分层表】**

| 指令 | 执行频率 | 触发条件 | 说明 |
|------|---------|---------|------|
| **G-6-1 句式节奏打断** | **每章必须** | 无条件触发 | 对流畅感影响最大，无豁免 |
| **G-6-2 感官描写替换** | **每章必须** | 无条件触发 | 抽象氛围词是AI感最明显标志，无豁免 |
| **G-6-3 对话潜台词** | **高情绪场景触发时执行** | 本章含有：决裂/告别/误解/表白/背叛等高情绪场景 | 无高情绪场景的章节（纯行动章/战斗推进章）可豁免 |
| **G-6-4 闲笔注入** | **每3章至少1次** | 距离上次闲笔已超过2章时强制触发 | 连续紧张章节中集中注入效果更好，不强制分散 |

> **注意**：G-6-3 和 G-6-4 的豁免须在章节自检报告中注明豁免原因（见下方自检输出格式）。

---

**【G-6-1】句式节奏打断**

每个段落内，句子长短必须不均匀。

检验方式：把段落里每句话的字数写出来——如果字数基本相同（比如全是20字左右），这段必须改。

执行规则：
- 每个段落至少有一句**极短句（≤5字，独立成行）**
- 每3个段落内，至少出现一次以下标点之一：**感叹号、省略号、破折号**
- 这三种标点不能集中在同一处，必须分散在不同的情绪节点

❌ AI感版（句式均匀，节奏平滑）：
> 他走进了那栋废弃的建筑，四周充满了阴森的气息，破旧的木板在脚下发出声响，让他感到一阵不安。

✅ 反AI感版（节奏有阻力）：
> 木板嘎吱了一声。
> 他停下来。
> 什么都没有——只有那声响还在耳朵里转，转了好几圈才散。

---

**【G-6-2】感官描写：禁用抽象形容词**

描写环境/氛围时，禁止使用以下类型的词汇作为主描写：
- ❌ 阴森恐怖、压抑、诡异、沉重、令人不安……

**必须用感官替代**：五种感官中至少覆盖三种，且每种感官的描写必须具体到物质层面。

| 感官 | 禁止写法 | 正确写法示例 |
|------|---------|-----------|
| 嗅觉 | "空气中弥漫着腐败的气息" | "霉味混着某种腥气——像是积了很久的雨水，又像是别的什么" |
| 听觉 | "四周寂静得令人心悸" | "木板在脚下嘎吱作响，但踩下去的那一秒之前有一段空白，那段空白更难受" |
| 触觉 | "感到一阵寒意" | "凉意顺着领口往里钻，钻到锁骨那里停住了，像一根手指按在那里没有松开" |
| 视觉 | "光线昏暗，看不清楚" | "唯一的光从门缝里进来，照出空气里浮着的灰，灰在转" |
| 味觉 | "嘴里发苦" | "嘴里有股铁腥味，他意识到自己一直在咬内侧的腮帮子" |

---

**【G-6-3】对话潜台词：禁止直给情绪**

在写决裂戏、告别戏、误解戏等高情绪场景时，禁止人物直接说出情绪词汇。

**禁止出现的直给写法**：
- "我恨你" / "我爱你" / "我失望了" / "你让我很痛苦" / "我再也不想见你"

**替代方式**：让人物说无关紧要的话，但用**一个具体的、非言语的行为**完成情绪传递。

执行规则：
- 高情绪场景结束时，离开的一方必须有一个**下意识的、与情绪相关的动作**
- 这个动作本身看起来无意义，但读者能感受到它的重量

示例：
> ❌ 直给版：
> "我再也不想看见你，"她说，"你毁了我。"
>
> ✅ 潜台词版：
> 她没有说话。只是把桌上那个杯子推到了他那边，然后拿起包走了。
> 走到门口，她停了一下——那枚他送的、她说过要扔掉的发簪，还别在头上。
> 她没有取下来。
> 门关上了。

---

**【G-6-4】闲笔注入：每章至少1处**

在紧张的情节推进中，强制插入**1个主角突然注意到的、与主线完全无关的细节**。

这个细节必须满足：
1. **足够"无聊"**：它对剧情没有任何推动作用，读者预期它接下来会有用，但不会有用
2. **足够真实**：它必须是人在高压情境下真实会注意到的那种荒诞微小的事
3. **一句话完成**：不展开，不解释，下一句立刻回到主线

示例（正确）：
> 反派站在高台上宣判他的死刑，声音洪亮，慷慨激昂——他的嘴角粘着一小块面包屑，从刚才就在那里，一直没掉。
>
> （下一句立刻继续主线）

示例（正确）：
> 他被逼到了墙角，四面是敌人，退无可退。这时候他突然注意到，左边那个拿刀的人，靴子上有一只蚂蚁，正在往上爬。

示例（禁止）：
> 他注意到地上有一朵花——这让他想起了童年，想起了那段逝去的岁月，想起了……
> （闲笔一旦展开就不是闲笔，是刻意的，失去了效果）

**【G-6】章节生成后四项快检**

| 指令 | 执行频率 | 检验问题 | 未通过处理 |
|------|---------|---------|---------|
| G-6-1 句式节奏 | **每章必须** | 随机抽取3个段落，是否每段都有极短句？感叹号/省略号/破折号是否分散出现？ | 找出最"流畅"的段落，强制插入停顿 |
| G-6-2 感官描写 | **每章必须** | 本章有无用"阴森""压抑"等抽象形容词代替感官描写？ | 找出该处，改写为≥3种感官的具体描述 |
| G-6-3 对话潜台词 | **高情绪场景触发** | 本章有无高情绪场景？人物是否直说了情绪词汇？ | 删除情绪词，替换为非言语行为+下意识动作；无高情绪场景→在自检报告注明豁免 |

### 【G-7】防模板化总则（全书持续监控）

> 规则是脚手架，不是枷锁。当规则本身开始产生"套路感"，规则就已经失效了。
> 本模块监控全书维度的重复使用，防止任何技巧因高频使用而变成读者可预判的模板。

**【连续不重复规则】**

以下元素若连续出现，强制执行替换策略：

| 元素类型 | 连续上限 | 超出处理 |
|---------|---------|---------|
| 钩子结尾意象（如：系统面板/感应到气息/神秘来信）| **连续3次** | 第4章强制更换意象类型（已有规则，重申）|
| 扰动卡牌（同一张）| **连续2次** | 第3次强制换牌（已有规则，重申）|
| 微小视角收束类型（如：连续用"不起眼的物件"）| **连续2次** | 第3次强制切换至其他4类之一 |
| 战斗余震类型（如：连续用"旁观者认知更新"）| **连续2次** | 第3次强制切换余震类型 |
| G-6-4 闲笔素材来源（如：连续用昆虫/小动物）| **连续2次** | 第3次切换至不同类别（物件/气味/人物的荒诞细节）|

**【类型用完即创造新类型】**

当一个类别的所有预设选项已被用尽（如6张扰动卡牌全部在本卷内使用过），**不得重复使用**，须在继续写作前创造新的选项：

```
触发条件：某一类型的所有选项已在本卷全部使用过≥1次

执行流程：
  1. 输出提示：「[扰动卡牌/余震类型/视角类型]已全部在本卷使用过。」
  2. 生成2-3个新选项（风格一致，但情境/情绪方向不同）
  3. 将新选项追加到对应模块，本章从新选项中选取
  4. 新选项同样适用连续不重复规则
```

**【模板化自检（每10章执行）】**

- [ ] 近10章的钩子类型分布是否均匀？是否某类型占比超过50%？
- [ ] 近10章的扰动卡牌是否存在偏好某1-2张的情况？
- [ ] 主角的"标志性行为"（毒舌反击/淡淡走开）是否在相近场景中重复出现超过3次？（重复本身是记忆点，但过多重复会变成梗）
- [ ] G-6闲笔素材是否已形成固定风格偏好？（若是，本阶段强制切换素材方向）

## 第三步：生成小说正文

### 3.1 写作策略

- **分章生成**：每次只生成1章
  - **铺垫章**：3000字（节奏稳，不拖沓，结尾必须有钩子）
  - **小爽章**：3200字（有1-2个打脸/爽点兑现）
  - **大爽章**：3500-4000字（主要打脸/突破/史诗时刻，爽感需要足够篇幅铺开）
  - **史诗章**：4000字（大战/终极对决，允许超长单次输出）

- **前置记忆**（每章生成前自动注入）：
  - 故事圣经核心要素（体系设定 + 人物卡 + 伏笔清单）
  - 最近3章详细摘要
  - 当前章节大纲（来自第二步细纲）
  - 本章主角的当前境界/战力
  - 本章钩子类型预设

**【跨Session状态快照（强制）】**

> 每次对话生成章节结束后，必须自动输出一份「当前状态快照」，供用户在新对话中粘贴恢复进度。
> 不输出快照 = 用户下次新开对话时，所有设定将从零重建，设定崩塌风险极高。

**快照输出模式：差异输出（Delta Mode）**

> 完整快照仅在第1章后输出一次。此后每章结束，**只输出本章发生了变化的字段**，未变化字段不重复输出。
> 若当章无任何字段变化（纯铺垫章），输出「本章无状态变更」即可，不输出空模板。

**第1章后输出完整快照（模板）**：

```
═══════════════ 状态快照（第X章结束）·完整版 ═══════════════
【主角当前状态】
  境界：[境界名称] · 已掌握能力：[列出]
  金手指当前解锁程度：[描述]
```

## 【Q-1】首订转化规范（前3章生死关）

> 起点读者在第3章结束时做出"是否订阅"的决定。前3章必须完成：同情建立 + 金手指激活 + 小翻盘预告。

### 首订三章节奏

| 章节 | 核心任务 | 必须完成的事 | 禁止的事 |
|------|---------|-----------|---------|
| **第1章** | 建立同情 + 激活金手指 | 屈辱场景（有名有姓的欺压者）+ 金手指觉醒（结尾） | 主角一开始就很强 / 纯背景介绍章 |
| **第2章** | 金手指初探 + 建立期待 | 主角初次使用金手指，能力初步显现，让读者感受到潜力 | 主角马上大爆发（要控制节奏，让读者期待后续）|
| **第3章** | 小翻盘预告 + 首订钩子 | 主角用金手指获得第一个小胜利，欺压者还没有受到应有报应（这是持续追更的动力）| 第3章就把第一卷的大反派打脸（太快了，读者没有积累）|

### 首订钩子设计要点

读者在第3章结尾应该产生两个想法：
1. "这个金手指真的太帅了，后面会怎么发展？"（体系期待）
2. "那个欺压他的人迟早要完蛋，我要看他们被打脸那一章"（仇恨绑定）

**硬性红线**（违反任何一条，整章重写）：

| 红线 | 要求 |
|------|------|
| 红线① | 第1章必须有具体的屈辱场景（模糊的"众人嘲笑"不算，要有具体事件+具体人物） |
| 红线② | 第1章结尾必须有金手指觉醒信号（不要求完整激活，但要给信号） |
| 红线③ | 第2章主角必须有"主动尝试"金手指的行为（不能被动等待） |
| 红线④ | 第3章必须有小翻盘或翻盘预告（主角依然处于弱势，但读者能看到希望） |
| 红线⑤ | 前3章的欺压者必须有名有姓，且第3章结束时仍未受到惩处（为后续打脸保留动力）|

## 【Q-2】钩子分类系统（起点版）

> 起点钩子的目标不是"番茄的每2秒刺激"，而是"让读者明天更新了第一时间来看"。

### 钩子类型与强度评级

| 钩子类型 | 写法 | 适用位置 | 强度 |
|---------|------|---------|------|
| **实力信息炸弹** | 揭示主角/反派的真实实力远超预期 | 章节结尾 | ★★★★★ |
| **身份揭露型** | 某角色的真实身份/关系被暗示或揭露 | 章节结尾 | ★★★★★ |
| **打脸前置** | 屈辱场景结束，结尾暗示"这笔账会清算" | 章节结尾 | ★★★★ |
| **体系悬念** | 金手指/修炼体系出现意外现象，没有解释 | 章节结尾 | ★★★★ |
| **势力介入** | 更大的势力/人物出现，格局突然扩大 | 章节结尾 | ★★★★★ |
| **生死危机** | 主角或重要配角面临真实危险 | 章节结尾 | ★★★★ |
| **伏笔触碰** | 前文埋的伏笔被轻轻触碰，读者想到了什么 | 章节结尾 | ★★★ |
| **行动悬念** | 主角做了一个看不懂的决定/行动，不解释 | 章节开头/结尾 | ★★★ |

**强制要求**：
- 每章结尾必须使用钩子，强度不低于★★★
- 前3章（首订关键期）每章钩子强度不低于★★★★★
- **禁止内心感慨收尾**：钩子必须来自外部事件（对话/能力显现/他人行动/物件出现）
- 每月末（月票高峰）规划1-2章大爽章+★★★★★钩子

### 结尾意象连续性约束

同一意象连续3章出现 = 违规，须更换：

| 章节 | 结尾意象/事件 | 连续次数 | 是否违规 |
|------|------------|---------|---------|
| 第N章 | 系统面板数字 | 第2次 | ✅ |
| 第N+1章 | 系统面板数字 | 第3次 | ✅ |
| 第N+2章 | 系统面板数字 | 第4次 | ❌ 须更换 |

## 【Q-3】打脸梯队设计规范

> 起点读者的爽感来自"层层递进的打脸"，每次打脸后有更大的敌人出现，让读者永远有"等着看下一次打脸"的期待。

### 打脸梯队结构

```
标准梯队（至少5级）：

第1级：身边的小人（同门/同学/邻居等）
  → 第一卷前半段打脸，建立"主角不是好欺负的"认知

第2级：本地势力（门派长老/公司高层/本城豪族等）
  → 第一卷后半段至第二卷，打破局部权力格局

第3级：地区顶级势力（宗门宗主/都市家族/地区BOSS等）
  → 第二卷主要打脸目标，让主角名声打出去

第4级：国家/大陆级势力（顶级宗门/财阀/国家机构等）
  → 第三卷，格局扩张到全局

第5级（终极）：隐藏大BOSS（世界真正的统治者/仇人的真实面目）
  → 终局卷，史诗兑现
```

### 反派设计规范（每级梯队）

| 要素 | 要求 |
|------|------|
| **屈辱场景** | 每个反派必须有≥1个具体的对主角的屈辱行为，且有旁观者在场 |
| **傲慢依据** | 反派为什么瞧不起主角？依据要在当时看起来合理（读者理解但愤怒）|
| **打脸时的在场者** | 打脸场景的旁观者 = 打脸力度，人越多越爽 |
| **打脸后的反应** | 反派的反应要写足（不信 → 震惊 → 恐惧 → 认输），不能一笔带过 |
| **下一级的引出** | 每个梯队打完，必须自然引出下一级的存在（让读者知道还有更大的） |

### 嘴替设计

每次打脸场景，最好有一个**嘴替角色**替读者说出他们想说的话：
- 可以是旁观者（说出主角有多强）
- 可以是反派的手下（质疑主角/提醒老大）
- 不能是主角本人（主角说自己厉害 = 自吹，效果减半）

### 【Q-3 附】反派威胁感建立协议（强制）

> 打脸爽感的前提是：**读者必须先真正怕过这个反派**。
> 一个没有威胁感的反派被打脸 = 踩蚂蚁，不是复仇。
> 威胁感不是靠嘴说出来的（"他是全城最强的人！"），是靠**读者亲眼看到他的手段**积累起来的。

**三阶段建立协议（每个第3梯队以上的反派必须执行）**：

| 阶段 | 时机 | 必须完成的事 | 禁止的事 |
|------|------|-----------|---------|
| **第一阶段：让旁人先怕他** | 反派正式登场前1-3章 | 通过其他角色的恐惧/传闻/过往战绩侧面建立威压——写别人见到他的反应，不要写他本人 | 主角第一次见面就能与他平视 |
| **第二阶段：让读者亲眼见识** | 反派首次出场章 | 反派必须有一个展示真实手段的高光时刻（对付的不是主角，而是第三方）——读者要有"主角要怎么赢？"的真实疑问 | 反派首场只会说狠话，无实际行动 |
| **第三阶段：打压主角建立仇恨** | 屈辱积累期 | 反派打压主角的行为必须有其内部逻辑（他为什么这样做是合理的），不能是纯蠢货 | 反派傲慢到让读者觉得他蠢——蠢反派被打脸没有爽感 |

**反派高光时刻设计规则**：
- 反派的高光必须针对一个读者有一定好感的角色（伤害无关路人不算）
- 高光行为要展示反派的**具体能力**，不是战力数字（"他一掌粉碎了X的护体金身"比"他的实力已超越X段境界"有效10倍）
- 高光结束后，必须有一个旁观者说出读者心里的话："主角遇到他……怎么可能赢？"

**反派崩塌节奏（打脸章）**：
```
"不信" → 2-3个来回，反派确认主角真的在挑战他
"震惊" → 主角展示真实实力的第一击，反派第一次动摇
"恐惧" → 反派意识到局势已经反转，开始真正的慌乱
"认输/崩溃" → 彻底落败，反应要写足，不能一句"他输了"带过
"余波" → 围观者的认知更新（这才是真正的"打脸"完成信号）
```

**禁止设计**：
- ❌ 反派突然变蠢来配合主角打脸（读者会出戏）
- ❌ 反派被打脸后立刻认怂跪地求饶（没有骨气的反派让打脸贬值）
- ❌ 反派高光戏份比主角打脸戏份多（威胁感和爽感要平衡）

## 【Q-4】标志性记忆点设计（起点版）

> 起点的标志性记忆点不同于搞笑无厘头，它是**主角气质的具象化载体**，是读者在书评区引用、对话里提及的东西。

### 类型与设计要求

| 类型 | 说明 | 示例 |
|------|------|------|
| **标志性台词** | 主角在关键时刻说的话，脱离上下文也有力量感 | "你配让我出手。" |
| **标志性行为** | 主角的招牌动作，一看到就知道"要出手了" | 捏碎手里的东西/淡淡看一眼/一句话不说直接走 |
| **标志性道具** | 和主角绑定的物件，在故事中多次出现并有不同含义 | 废旧的令牌/一把锁/父母留下的东西 |

### 三阶段设计（建立→变形→升华）

**第一阶段（第一卷）**：以功能性方式出现，读者觉得有意思但不觉得有深意

**第二阶段（中间各卷）**：同样的元素在更大的场景下出现，重量变了

**第三阶段（终局卷）**：最后一次出现，承载整本书的主题，读者产生情感叠加

**在故事圣经中，标志性记忆点必须单独列出：**

| 元素 | 类型 | 建立章节 | 变形节点（每卷） | 升华章节 | 主题联系 |
|------|------|---------|----------------|---------|---------|
| [台词/行为/物件] | — | 第X章 | 卷二·第Y章 | 终局·第Z章 | — |

## 【Q-5】月票与订阅节奏规划

> 起点的商业逻辑需要作者主动规划"爆发节点"，不是随机写到哪里就是哪里。

### 节奏规划框架

```
每月节奏模型：

  月初（1-5号）：承接上月末钩子，延续热度
  月中（10-20号）：稳定更新，铺垫为主，偶发小爽章维持留存
  月末（25-31号）：★★★★★大爽章爆发，冲击月票榜
```

### 卷末节奏规划

每卷结束前必须完成：
1. **该卷主打脸兑现**（读者等了这么久，要给爽感回报）
2. **更大格局开启**（让读者期待下一卷）
3. **感情线里程碑达成**（小甜点，稳住老读者）

### 【Q-5 附】世界格局扩张节奏

> 起点顶流书的核心爽感之一是"主角的世界越来越大"——读者随着主角一起，不断发现原来以为的天花板只是更大世界的地板。
> 但格局扩张不能随意，扩得太快读者跟不上，扩得太慢读者觉得憋。

**格局扩张四原则**：

1. **每卷至少扩张一级**：每卷结束，主角的活动范围/对手层级必须比这卷开始时高一级
2. **天花板要提前埋设**：主角当前圈子的「天花板人物」要在正式对决的2-3卷前就出现过（哪怕只是一个名字、一个传闻），让读者能感知到上面还有更大的世界
3. **格局扩张要有"震撼时刻"**：每次格局扩张，必须有一个让读者觉得"原来如此，之前的一切都只是序章"的揭示场景
4. **主角要有"配得上"的感知**：格局扩张时，主角对新世界的反应要是克制的、有底气的（不是惊慌失措），哪怕他目前还不够强

**格局扩张预规划表**（故事圣经中必须填写）：

| 卷次 | 本卷活动范围 | 顶级对手层级 | 卷末揭示的「更大世界」信号 | 天花板人物（名字+身份）|
|------|-----------|-----------|----------------------|-------------------|
| 第一卷 | — | — | — | — |
| 第二卷 | — | — | — | — |
| 第三卷 | — | — | — | — |

**禁止设计**：
- ❌ 突然空降一个之前完全没有铺垫的超强势力（读者会感到剧情失控）
- ❌ 格局扩张时主角的反应是"天哪太厉害了我怎么可能赢"（王者气质不允许）

## 【Q-8】书评区话题设计规范

> 起点留存公式里有一项「社区互动感」——这不是靠运气产生的，是靠**主动设计**出来的。
> 书评区活跃 → 读者粘性高 → 月票/打赏/订阅联动上升。
> 核心逻辑：**每隔3-5章，主动在正文里埋一个「让读者想去书评区讨论的话题」**。

### 书评区话题类型与设计方法

| 话题类型 | 触发方式 | 示例 |
|---------|---------|------|
| **理论研究型** | 金手指/体系出现一个模糊现象，没有解释 | 系统播报了一个奇怪数值，主角没有在意——但读者会在意 |
| **身份猜测型** | 某个神秘角色出现，背景刻意模糊 | 一个对主角异常了解的陌生人，说完一句话就走了 |
| **伏笔预判型** | 某个细节在前文出现了两次，第三次还没出现 | 读者会在书评区问"上次那个XX是什么意思？" |
| **阵营争论型** | 一个角色的行为让读者对他的立场产生分歧 | "他到底是敌是友？" 书评区必然分成两派 |
| **时间线谜题型** | 叙事中插入一个时间线不对的细节，不解释 | "等等，他怎么可能在这里，他上章不是在另一个地方？" |

### 执行规则

- 每3-5章必须有至少1个书评区话题触发点
- 话题必须是**主动设计的**，不是偶然产生的
- 话题的答案不能在同章揭晓（至少3章后才能有官方解释）
- 话题密度不能太高：连续3章都有悬念 = 读者信息过载，产生焦虑而非期待

### 书评区话题自检（每5章执行）

- [ ] 近5章内是否有≥1个书评区话题触发点？
- [ ] 该话题的答案是否已有规划（不是随手埋的烂尾悬念）？
- [ ] 话题是否足够具体可讨论（不是泛泛的"他将来会怎样"，而是"这个细节是什么意思"）？

## 【Q-6】水章管理规范

> 起点长篇小说不可避免有铺垫章（俗称"水章"），但水章管理不好会掉订阅。

### 水章分级

| 级别 | 特征 | 处理方式 |
|------|------|---------|
| **一级水章** | 纯日常/纯对话，无推进 | 禁止出现，整章重写 |
| **二级水章** | 有推进但节奏慢，无明显爽点 | 允许，但必须有★★★以上钩子 |
| **三级铺垫章** | 有世界观深化/人物关系推进/伏笔埋设 | 允许，必须有★★★钩子，且铺垫目标明确 |

**水章硬性规则**：
- 连续铺垫章不得超过3章（第4章必须有爽点兑现）
- 每个铺垫章必须有"可见的进展"（哪怕只是一个小的世界观细节被揭示）
- 水章结尾必须有对下一章的期待感（哪怕是轻微的钩子）

## 【Q-7】主角低谷期设计规范（弃坑高危区）

> 低谷期是长篇小说的必要结构——没有低谷，高潮就没有对比。
> 但低谷期是**弃坑率最高的区段**，处理不好读者直接走人。
> 核心原则：**低谷不是主角变弱，是主角面对更大的问题而暂时没有解法。**

### 低谷期设计五条硬规则

| 规则 | 要求 | 违反后果 |
|------|------|---------|
| **① 低谷有明确上限** | 低谷连续不超过5章，第6章必须出现反弹锚点（哪怕只是一个小希望信号）| 读者感知不到尽头，直接弃坑 |
| **② 禁止人格崩塌** | 低谷期主角可以受挫、沉默、痛苦，但禁止：无脑抱怨 / 自怨自艾超过1章 / 行为变得懦弱或反智 | 主角形象崩塌，读者失去信任感 |
| **③ 低谷原因要"值得"** | 让主角陷入低谷的原因必须让读者感受到"这个困难确实超出了他目前的能力范围"，不能是人为降智 | 读者感到被愚弄 |
| **④ 必须埋反弹锚点** | 低谷期必须同步埋入至少1个"反弹锚点"——一个让读者知道主角迟早会翻盘的细节或伏笔（可以很隐晦，但必须有）| 读者没有追更的理由 |
| **⑤ 低谷结束要超额兑现** | 低谷结束后的第一次翻盘，其爽感必须明显超过低谷前的水平——"憋了这么久，就是为了这一刻" | 低谷付出的耐心没有回报，读者觉得亏 |

### 低谷期节奏模型

```
低谷期标准结构（最多5章）：

  第1章 → 低谷触发：主角遭遇真正超出能力范围的打击（不是被打败，是"当前的他解决不了"）
  第2章 → 压力加码：同时从另一个方向施压（让读者感受到"屋漏偏逢连夜雨"的窒息感）
  第3章 → 沉淀与转机萌芽：主角沉默/思考，反弹锚点在此章出现（金手指新线索/旧人重新出现/关键信息揭露）
  第4章 → 主动求变：主角做出一个让读者觉得"对，就这么干"的决定——不是靠运气，是靠他本人
  第5章 → 反弹起点：第一个小胜利，明确告知读者：他回来了
```

### 低谷期禁用写法

- ❌ "他躺在地上，觉得一切都没有意义"（超过半章的消沉独白）
- ❌ 主角在低谷期做了一个让读者觉得蠢的选择（降智设计）
- ❌ 低谷靠外力解决：另一个人突然来救他，他自己什么都没做
- ❌ 低谷结束后，主角的实力只是恢复到低谷前的水平（没有"憋大招"的感觉）

### 低谷期自检（进入低谷期前必须确认）

- [ ] 低谷的触发原因是否让读者感到"确实超出了他的能力"（而非主角变蠢）？
- [ ] 本次低谷计划持续几章？是否≤5章？
- [ ] 反弹锚点已规划在哪一章、以何种方式出现？
- [ ] 低谷结束后的翻盘爽感，是否明显高于低谷前的平均水平？

### 3.2 每章生成后自检

**【首订关键期检查（仅第1-3章执行，优先级最高）】**

- [ ] 第1章：是否有具体的屈辱场景（有名有姓的欺压者，有旁观者）
- [ ] 第1章：结尾是否有金手指觉醒信号
- [ ] 第2章：主角是否有主动尝试金手指的行为
- [ ] 第3章：是否有小翻盘或翻盘预告（主角依然弱势，但读者看到希望）
- [ ] 前3章：欺压者是否仍未受到惩处（保留后续打脸动力）
- [ ] 前3章钩子强度：每章是否达到★★★★★

**【常规检查（每章执行）】**

- [ ] 字数是否达标（铺垫章3000 / 小爽章3200 / 大爽章3500-4000 / 史诗章≤4500）
- [ ] 章节结尾钩子类型是否符合细纲标注，强度是否≥★★★
- [ ] 结尾是否为外部信号（禁止内心感慨收尾）
- [ ] 结尾意象是否连续使用超过3章（超过=本章强制更换）
- [ ] 若本章是打脸章：屈辱积累是否足够？翻盘是否有旁观者？反派反应是否写足？
- [ ] 若本章是突破章：是否有"卡关"在前？突破过程是否有画面感？
- [ ] 主角性格是否保持一致（冷静克制，毒舌偶发，对敌无情，对己方有底线）
- [ ] 幽默是否为克制型（反差/毒舌），是否有无厘头搞笑场景（有则删除）
- [ ] 若是大场面：是否完成"危机→应对→反转→余波"四步？
- [ ] 若是大场面：余波段是否执行了**微小视角收束**？视角类型是否与主角灵魂拷问答案有内在呼应？
- [ ] 若是大爽章/史诗章：是否已使用扰动卡牌并融入细纲？扰动结果是否在章节中体现？
- [ ] 升级节奏：距离上次"变强信号"是否已超过5章（超过则本章必须给信号）
- [ ] 未出现设定文档之外的即兴设定（如有，提示用户决定是否保留）
- [ ] 体系数据一致性（境界/战力数值与故事圣经一致）
- [ ] 伏笔管理：未提前揭露计划外伏笔
- [ ] **【G-6-1】随机抽取3段，每段是否有极短句？感叹号/省略号/破折号是否分散出现？**
- [ ] **【G-6-2】本章是否出现"阴森""压抑""诡异"等抽象氛围词？（出现=找到并改写为≥3种感官）**
- [ ] **【G-6-3】本章高情绪场景：人物是否直说了情绪词汇？（出现=删除，替换为非言语行为）**
- [ ] **【G-6-4】本章是否有≥1处"无聊但真实"的无关闲笔细节？（无=找最紧张段落强制插入）**
- [ ] **【Q-7低谷期】若本章处于低谷期：是否已超过5章？本章是否出现主角人格崩塌/无脑抱怨？反弹锚点是否已埋入？**
- [ ] **【Q-8书评区】近5章内是否有书评区话题触发点？该悬念是否有预设解答章节？**

### 【3.2-报告】每章自检报告输出格式（章节生成后强制输出）

> 自检不只是内部检查——每章生成后，必须输出一段简短的**可见自检报告**，让用户能快速判断本章质量。
> 报告格式：条目分类 + 通过/未通过 + 未通过原因（已修复还是待处理）。

```
━━━━━━━━━━━━━ 第X章自检报告 ━━━━━━━━━━━━━

【P0铁律检查】
  ✅ 钩子强度：★★★★（≥★★★，通过）
  ✅ 连续铺垫计数：本章为第2章连续铺垫（未超3章限制）
  [若首订期] ✅/❌ 首订红线①-⑤：[逐条标注]

【读者情绪检查（P1）】
  ✅/❌ 战斗情绪债：[通过 / 未通过·已修复：补充了「旁观者担心」积累]
  ✅/❌ 屈辱积累充分度：[通过 / 待处理·原因：]

【G-6 反AI感执行】
  ✅ G-6-1 句式节奏：已执行（随机抽检3段，均含极短句）
  ✅ G-6-2 感官描写：已执行（未发现抽象氛围词）
  ✅/豁免 G-6-3 对话潜台词：[已执行 / 豁免·原因：本章无高情绪场景]
  ✅/待触发 G-6-4 闲笔注入：[已注入第X段 / 距上次2章，下章强制触发]

【防模板检查】
  ✅/⚠️ 钩子意象连续：[正常 / 警告：已连续2次使用「系统面板」，下章须更换]
  ✅/⚠️ 扰动卡/余震连续：[正常 / 警告：]

【总结】
  本章状态：✅ 全部通过 / ⚠️ X项待注意（已修复X项，剩余X项下章处理）
  下章注意事项：[1-2句提示]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**输出规则**：
- 报告在章节正文**之后**输出，用分隔线明确区分正文与报告
- ✅ = 通过；❌ = 未通过（须注明是否已修复）；⚠️ = 警告（接近上限）；豁免/待触发 = 频率分层允许的例外，须注明原因
- 若某条目与本章类型无关（如打脸章检查项但本章非打脸章），标注 `—（不适用）` 而非留空

### 3.3 阶段性一致性检查（每5章执行）

自动输出报告：
- 主角当前境界与细纲进度是否一致
- 打脸梯队进度（当前在第几级，屈辱积累是否足够）
- 伏笔埋设与中期触碰进度
- 感情线里程碑进度
- **【情感债盘点】**：主角当前欠谁的债？谁欠主角的债？本阶段有无新增或偿还？
- **【扰动卡执行记录】**：近5章内是否有大爽章/史诗章？是否已使用扰动卡，结果如何？
- 月票节奏：近5章内是否有大爽章或需要补充
- 连续铺垫章数量警告（超过3章须立即插入爽点）

### 3.4 全书完成后最终报告

- 合并所有章节，检查章节间过渡
- 生成**打脸梯队完成报告**（每级梯队的屈辱积累 vs 打脸兑现是否平衡）
- 生成**伏笔回收对照表**（未回收的伏笔列出）
- 生成**升级节奏报告**（每10章的战力/境界变化）
- 生成**钩子效果统计**（各章强度，低于★★★的章节标红）
- 生成**【Q-4】标志性记忆点三阶段完整性确认**

## 钩子快速参考卡

**测试一：首订测试**
把前3章的摘要发给一个没看过网文的朋友，对方是否想知道"那个欺负主角的人最后怎么了"？
- 想知道 → 首订钩子有效
- 不想知道 → 屈辱积累不够，或金手指潜力没展示到位

**测试二：追更测试**
读者看完这章，明天更新了会不会第一时间来看？
- ★★★★★：手机铃声未响就已经打开了
- ★★★★：更新提醒来了立刻点进来
- ★★★：有空就看
- ★★：记起来才看
- ★：不一定记得

**测试三：书评区测试**
读者读完这章，是否有冲动在书评区写"作者你赶快写X章那个谁被打脸"？
- 有 → 仇恨绑定成功，后续打脸的期待已经建立
- 没有 → 屈辱不够深，或打脸时机选错了
